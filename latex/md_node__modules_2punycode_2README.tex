 Punycode.\+js is a robust Punycode converter that fully complies to \href{https://tools.ietf.org/html/rfc3492}{\texttt{ RFC 3492}} and \href{https://tools.ietf.org/html/rfc5891}{\texttt{ RFC 5891}}.

This Java\+Script library is the result of comparing, optimizing and documenting different open-\/source implementations of the Punycode algorithm\+:


\begin{DoxyItemize}
\item \href{https://tools.ietf.org/html/rfc3492\#appendix-C}{\texttt{ The C example code from RFC 3492}}
\item \href{http://opensource.apple.com/source/ICU/ICU-400.42/icuSources/common/punycode.c}{\texttt{ {\ttfamily punycode.\+c} by {\itshape Markus W. Scherer} (IBM)}}
\item \href{https://github.com/bnoordhuis/punycode/blob/master/punycode.c}{\texttt{ {\ttfamily punycode.\+c} by {\itshape Ben Noordhuis}}}
\item \href{http://stackoverflow.com/questions/183485/can-anyone-recommend-a-good-free-javascript-for-punycode-to-unicode-conversion/301287\#301287}{\texttt{ Java\+Script implementation by {\itshape some}}}
\item \href{https://github.com/joyent/node/blob/426298c8c1c0d5b5224ac3658c41e7c2a3fe9377/lib/punycode.js}{\texttt{ {\ttfamily punycode.\+js} by {\itshape Ben Noordhuis}}} (note\+: \href{https://github.com/joyent/node/issues/2072}{\texttt{ not fully compliant}})
\end{DoxyItemize}

This project was \href{https://github.com/joyent/node/blob/master/lib/punycode.js}{\texttt{ bundled}} with Node.\+js from \href{https://github.com/joyent/node/compare/975f1930b1...61e796decc}{\texttt{ v0.\+6.\+2+}} until \href{https://github.com/nodejs/node/pull/7941}{\texttt{ v7}} (soft-\/deprecated).

Converts a Punycode string representing a domain name or an email address to Unicode. Only the Punycoded parts of the input will be converted, i.\+e. it doesn’t matter if you call it on a string that has already been converted to Unicode.

\DoxyCodeLine{punycode.toUnicode('xn-\/-\/maana-\/pta.com');}
\DoxyCodeLine{//\ →\ 'mañana.com'}
\DoxyCodeLine{punycode.toUnicode('xn-\/-\/-\/-\/dqo34k.com');}
\DoxyCodeLine{//\ →\ '☃-\/⌘.com'}
\DoxyCodeLine{}
\DoxyCodeLine{//\ decode\ email\ addresses}
\DoxyCodeLine{punycode.toUnicode('джумла@xn-\/-\/p-\/8sbkgc5ag7bhce.xn-\/-\/ba-\/lmcq');}
\DoxyCodeLine{//\ →\ 'джумла@джpумлатест.bрфa'}

\hypertarget{md_node__modules_2punycode_2README_autotoc_md22445}{}\doxysubsubsection{\texorpdfstring{$<$tt$>$punycode.\+ucs2.\+decode(string)$<$/tt$>$}{<tt>punycode.ucs2.decode(string)</tt>}}\label{md_node__modules_2punycode_2README_autotoc_md22445}
Creates an array containing the numeric code point values of each Unicode symbol in the string. While \href{https://mathiasbynens.be/notes/javascript-encoding}{\texttt{ Java\+Script uses UCS-\/2 internally}}, this function will convert a pair of surrogate halves (each of which UCS-\/2 exposes as separate characters) into a single code point, matching UTF-\/16.


\begin{DoxyCode}{0}
\DoxyCodeLine{punycode.ucs2.decode('abc');}
\DoxyCodeLine{//\ →\ [0x61,\ 0x62,\ 0x63]}
\DoxyCodeLine{//\ surrogate\ pair\ for\ U+1D306\ TETRAGRAM\ FOR\ CENTRE:}
\DoxyCodeLine{punycode.ucs2.decode('\(\backslash\)uD834\(\backslash\)uDF06');}
\DoxyCodeLine{//\ →\ [0x1D306]}

\end{DoxyCode}
