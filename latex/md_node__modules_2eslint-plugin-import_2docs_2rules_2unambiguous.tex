\chapter{import/unambiguous}
\hypertarget{md_node__modules_2eslint-plugin-import_2docs_2rules_2unambiguous}{}\label{md_node__modules_2eslint-plugin-import_2docs_2rules_2unambiguous}\index{import/unambiguous@{import/unambiguous}}

\Hypertarget{md_node__modules_2eslint-plugin-import_2docs_2rules_2unambiguous_autotoc_md11026}%


Warn if a {\ttfamily module} could be mistakenly parsed as a {\ttfamily script} by a consumer leveraging \href{https://github.com/nodejs/node-eps/blob/HEAD/002-es-modules.md\#32-determining-if-source-is-an-es-module}{\texttt{ Unambiguous Java\+Script Grammar}} to determine correct parsing goal.

Will respect the \href{https://eslint.org/docs/user-guide/configuring\#specifying-parser-options}{\texttt{ {\ttfamily parser\+Options.\+source\+Type}}} from ESLint config, i.\+e. files parsed as {\ttfamily script} per that setting will not be reported.

This plugin uses \href{https://github.com/nodejs/node-eps/blob/HEAD/002-es-modules.md\#32-determining-if-source-is-an-es-module}{\texttt{ Unambiguous Java\+Script Grammar}} internally to decide whether dependencies should be parsed as modules and searched for exports matching the {\ttfamily import}ed names, so it may be beneficial to keep this rule on even if your application will run in an explicit {\ttfamily module}-\/only environment.\hypertarget{md_node__modules_2eslint-plugin-import_2docs_2rules_2unambiguous_autotoc_md11027}{}\doxysection{\texorpdfstring{Rule Details}{Rule Details}}\label{md_node__modules_2eslint-plugin-import_2docs_2rules_2unambiguous_autotoc_md11027}
For files parsed as {\ttfamily module} by ESLint, the following are valid\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{import\ 'foo'}
\DoxyCodeLine{function\ x()\ \{\ return\ 42\ \}}

\end{DoxyCode}



\begin{DoxyCode}{0}
\DoxyCodeLine{export\ function\ x()\ \{\ return\ 42\ \}}

\end{DoxyCode}



\begin{DoxyCode}{0}
\DoxyCodeLine{(function\ x()\ \{\ return\ 42\ \})()}
\DoxyCodeLine{export\ \{\}\ //\ simple\ way\ to\ mark\ side-\/effects-\/only\ file\ as\ 'module'\ without\ any\ imports/exports}

\end{DoxyCode}


...whereas the following file would be reported\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{(function\ x()\ \{\ return\ 42\ \})()}

\end{DoxyCode}
\hypertarget{md_node__modules_2eslint-plugin-import_2docs_2rules_2unambiguous_autotoc_md11028}{}\doxysection{\texorpdfstring{When Not To Use It}{When Not To Use It}}\label{md_node__modules_2eslint-plugin-import_2docs_2rules_2unambiguous_autotoc_md11028}
If your application environment will always know via \href{https://github.com/nodejs/node-eps/issues/13}{\texttt{ some other means}} how to parse, regardless of syntax, you may not need this rule.

Remember, though, that this plugin uses this strategy internally, so if you were to {\ttfamily import} from a module with no {\ttfamily import}s or {\ttfamily export}s, this plugin would not report it as it would not be clear whether it should be considered a {\ttfamily script} or a {\ttfamily module}.\hypertarget{md_node__modules_2eslint-plugin-import_2docs_2rules_2unambiguous_autotoc_md11029}{}\doxysection{\texorpdfstring{Further Reading}{Further Reading}}\label{md_node__modules_2eslint-plugin-import_2docs_2rules_2unambiguous_autotoc_md11029}
