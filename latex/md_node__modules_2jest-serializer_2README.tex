 Module for serializing and deserializing object into memory and disk. By default, the {\ttfamily v8} implementations are used, but if not present, it defaults to {\ttfamily JSON} implementation. Both serializers have the advantage of being able to serialize {\ttfamily Map}, {\ttfamily Set}, {\ttfamily undefined}, {\ttfamily NaN}, etc, although the JSON one does it through a replacer/reviver.\hypertarget{md_node__modules_2jest-serializer_2README_autotoc_md17522}{}\doxysection{\texorpdfstring{Install}{Install}}\label{md_node__modules_2jest-serializer_2README_autotoc_md17522}

Three kinds of API groups are exposed\+:\hypertarget{md_node__modules_2jest-serializer_2README_autotoc_md17524}{}\doxysubsection{\texorpdfstring{In-\/memory serialization\+: $<$tt$>$serialize$<$/tt$>$ and $<$tt$>$deserialize$<$/tt$>$}{In-memory serialization: <tt>serialize</tt> and <tt>deserialize</tt>}}\label{md_node__modules_2jest-serializer_2README_autotoc_md17524}
This set of functions take or return a {\ttfamily Buffer}. All the process happens in memory. This is useful when willing to transfer over HTTP, TCP or via UNIX pipes.

\hypertarget{md_node__modules_2jest-serializer_2README_autotoc_md17525}{}\doxysubsection{\texorpdfstring{Synchronous persistent filesystem\+: $<$tt$>$read\+File\+Sync$<$/tt$>$ and $<$tt$>$write\+File\+Sync$<$/tt$>$}{Synchronous persistent filesystem: <tt>readFileSync</tt> and <tt>writeFileSync</tt>}}\label{md_node__modules_2jest-serializer_2README_autotoc_md17525}
