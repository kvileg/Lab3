This plugin uses \href{https://github.com/terser/terser}{\texttt{ terser}} to minify/minimize your Java\+Script.\hypertarget{md_node__modules_2terser-webpack-plugin_2README_autotoc_md27957}{}\doxysubsection{\texorpdfstring{Getting Started}{Getting Started}}\label{md_node__modules_2terser-webpack-plugin_2README_autotoc_md27957}
Webpack v5 comes with the latest {\ttfamily terser-\/webpack-\/plugin} out of the box. If you are using Webpack v5 or above and wish to customize the options, you will still need to install {\ttfamily terser-\/webpack-\/plugin}. Using Webpack v4, you have to install {\ttfamily terser-\/webpack-\/plugin} v4.

To begin, you\textquotesingle{}ll need to install {\ttfamily terser-\/webpack-\/plugin}\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{npm\ install\ terser-\/webpack-\/plugin\ -\/-\/save-\/dev}

\end{DoxyCode}

\begin{DoxyCode}{0}
\DoxyCodeLine{yarn\ add\ -\/D\ terser-\/webpack-\/plugin}

\end{DoxyCode}

\begin{DoxyCode}{0}
\DoxyCodeLine{pnpm\ add\ -\/D\ terser-\/webpack-\/plugin}

\end{DoxyCode}

\texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*}\+Works only with {\ttfamily source-\/map}, {\ttfamily inline-\/source-\/map}, {\ttfamily hidden-\/source-\/map} and {\ttfamily nosources-\/source-\/map} values for the \href{https://webpack.js.org/configuration/devtool/}{\texttt{ {\ttfamily devtool}}} option.\texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*}

Why?


\begin{DoxyItemize}
\item {\ttfamily eval} wraps modules in {\ttfamily eval("{}string"{})} and the minimizer does not handle strings.
\item {\ttfamily cheap} has not column information and minimizer generate only a single line, which leave only a single mapping.
\end{DoxyItemize}
