Forbid modules to have too many dependencies ({\ttfamily import} or {\ttfamily require} statements).

This is a useful rule because a module with too many dependencies is a code smell, and usually indicates the module is doing too much and/or should be broken up into smaller modules.

Importing multiple named exports from a single module will only count once (e.\+g. `import \{x, y, z\} from './foo\textquotesingle{}\`{} will only count as a single dependency).\hypertarget{md_node__modules_2eslint-plugin-import_2docs_2rules_2max-dependencies_autotoc_md10851}{}\doxysection{\texorpdfstring{Options}{Options}}\label{md_node__modules_2eslint-plugin-import_2docs_2rules_2max-dependencies_autotoc_md10851}
