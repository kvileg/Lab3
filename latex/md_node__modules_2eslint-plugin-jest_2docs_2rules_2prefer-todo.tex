 When test cases are empty then it is better to mark them as {\ttfamily test.\+todo} as it will be highlighted in the summary output.\hypertarget{md_node__modules_2eslint-plugin-jest_2docs_2rules_2prefer-todo_autotoc_md11473}{}\doxysection{\texorpdfstring{Rule details}{Rule details}}\label{md_node__modules_2eslint-plugin-jest_2docs_2rules_2prefer-todo_autotoc_md11473}
This rule triggers a warning if empty test cases are used without \textquotesingle{}test.\+todo\textquotesingle{}.


\begin{DoxyCode}{0}
\DoxyCodeLine{test('i\ need\ to\ write\ this\ test');}

\end{DoxyCode}
\hypertarget{md_node__modules_2eslint-plugin-jest_2docs_2rules_2prefer-todo_autotoc_md11474}{}\doxysubsection{\texorpdfstring{Default configuration}{Default configuration}}\label{md_node__modules_2eslint-plugin-jest_2docs_2rules_2prefer-todo_autotoc_md11474}
The following pattern is considered warning\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{test('i\ need\ to\ write\ this\ test');\ //\ Unimplemented\ test\ case}
\DoxyCodeLine{test('i\ need\ to\ write\ this\ test',\ ()\ =>\ \{\});\ //\ Empty\ test\ case\ body}
\DoxyCodeLine{test.skip('i\ need\ to\ write\ this\ test',\ ()\ =>\ \{\});\ //\ Empty\ test\ case\ body}

\end{DoxyCode}


The following pattern is not warning\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{test.todo('i\ need\ to\ write\ this\ test');}

\end{DoxyCode}
