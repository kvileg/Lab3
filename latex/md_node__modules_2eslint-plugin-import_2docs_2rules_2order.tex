Enforce a convention in the order of {\ttfamily require()} / {\ttfamily import} statements.

With the \`{}groups\`{} option set to {\ttfamily \mbox{[}"{}builtin"{}, "{}external"{}, "{}internal"{}, "{}parent"{}, "{}sibling"{}, "{}index"{}, "{}object"{}, "{}type"{}\mbox{]}} the order is as shown in the following example\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{//\ 1.\ node\ "{}builtin"{}\ modules}
\DoxyCodeLine{import\ fs\ from\ 'fs';}
\DoxyCodeLine{import\ path\ from\ 'path';}
\DoxyCodeLine{//\ 2.\ "{}external"{}\ modules}
\DoxyCodeLine{import\ \_\ from\ 'lodash';}
\DoxyCodeLine{import\ chalk\ from\ 'chalk';}
\DoxyCodeLine{//\ 3.\ "{}internal"{}\ modules}
\DoxyCodeLine{//\ (if\ you\ have\ configured\ your\ path\ or\ webpack\ to\ handle\ your\ internal\ paths\ differently)}
\DoxyCodeLine{import\ foo\ from\ 'src/foo';}
\DoxyCodeLine{//\ 4.\ modules\ from\ a\ "{}parent"{}\ directory}
\DoxyCodeLine{import\ foo\ from\ '../foo';}
\DoxyCodeLine{import\ qux\ from\ '../../foo/qux';}
\DoxyCodeLine{//\ 5.\ "{}sibling"{}\ modules\ from\ the\ same\ or\ a\ sibling's\ directory}
\DoxyCodeLine{import\ bar\ from\ './bar';}
\DoxyCodeLine{import\ baz\ from\ './bar/baz';}
\DoxyCodeLine{//\ 6.\ "{}index"{}\ of\ the\ current\ directory}
\DoxyCodeLine{import\ main\ from\ './';}
\DoxyCodeLine{//\ 7.\ "{}object"{}-\/imports\ (only\ available\ in\ TypeScript)}
\DoxyCodeLine{import\ log\ =\ console.log;}
\DoxyCodeLine{//\ 8.\ "{}type"{}\ imports\ (only\ available\ in\ Flow\ and\ TypeScript)}
\DoxyCodeLine{import\ type\ \{\ Foo\ \}\ from\ 'foo';}

\end{DoxyCode}


Unassigned imports are ignored, as the order they are imported in may be important.

This rule supports the following options\+:\hypertarget{md_node__modules_2eslint-plugin-import_2docs_2rules_2order_autotoc_md11012}{}\doxysubsection{\texorpdfstring{$<$tt$>$groups\+: \mbox{[}array\mbox{]}$<$/tt$>$}{<tt>groups: [array]</tt>}}\label{md_node__modules_2eslint-plugin-import_2docs_2rules_2order_autotoc_md11012}
How groups are defined, and the order to respect. {\ttfamily groups} must be an array of {\ttfamily string} or \mbox{[}{\ttfamily string}\mbox{]}. The only allowed {\ttfamily string}s are\+: {\ttfamily "{}builtin"{}}, {\ttfamily "{}external"{}}, {\ttfamily "{}internal"{}}, {\ttfamily "{}unknown"{}}, {\ttfamily "{}parent"{}}, {\ttfamily "{}sibling"{}}, {\ttfamily "{}index"{}}, {\ttfamily "{}object"{}}, {\ttfamily "{}type"{}}. The enforced order is the same as the order of each element in a group. Omitted types are implicitly grouped together as the last element. Example\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{[}
\DoxyCodeLine{\ \ 'builtin',\ //\ Built-\/in\ types\ are\ first}
\DoxyCodeLine{\ \ ['sibling',\ 'parent'],\ //\ Then\ sibling\ and\ parent\ types.\ They\ can\ be\ mingled\ together}
\DoxyCodeLine{\ \ 'index',\ //\ Then\ the\ index\ file}
\DoxyCodeLine{\ \ 'object',}
\DoxyCodeLine{\ \ //\ Then\ the\ rest:\ internal\ and\ external\ type}
\DoxyCodeLine{]}

\end{DoxyCode}


The default value is {\ttfamily \mbox{[}"{}builtin"{}, "{}external"{}, "{}parent"{}, "{}sibling"{}, "{}index"{}\mbox{]}}.

\hypertarget{md_node__modules_2eslint-plugin-import_2docs_2rules_2order_autotoc_md11014}{}\doxysubsection{\texorpdfstring{$<$tt$>$distinct\+Group\+: \mbox{[}boolean\mbox{]}$<$/tt$>$}{<tt>distinctGroup: [boolean]</tt>}}\label{md_node__modules_2eslint-plugin-import_2docs_2rules_2order_autotoc_md11014}
This changes how {\ttfamily path\+Groups\mbox{[}\mbox{]}.position} affects grouping. The property is most useful when {\ttfamily newlines-\/between} is set to {\ttfamily always} and at least 1 {\ttfamily path\+Groups} entry has a {\ttfamily position} property set.

By default, in the context of a particular {\ttfamily path\+Group} entry, when setting {\ttfamily position}, a new "{}group"{} will silently be created. That is, even if the {\ttfamily group} is specified, a newline will still separate imports that match that {\ttfamily pattern} with the rest of the group (assuming {\ttfamily newlines-\/between} is {\ttfamily always}). This is undesirable if your intentions are to use {\ttfamily position} to position {\itshape within} the group (and not create a new one). Override this behavior by setting {\ttfamily distinct\+Group} to {\ttfamily false}; this will keep imports within the same group as intended.

Note that currently, {\ttfamily distinct\+Group} defaults to {\ttfamily true}. However, in a later update, the default will change to {\ttfamily false}

The default value is {\ttfamily \mbox{[}"{}builtin"{}, "{}external"{}, "{}object"{}\mbox{]}}.\hypertarget{md_node__modules_2eslint-plugin-import_2docs_2rules_2order_autotoc_md11016}{}\doxysubsection{\texorpdfstring{$<$tt$>$newlines-\/between\+: \mbox{[}ignore\texorpdfstring{$\vert$}{|}always\texorpdfstring{$\vert$}{|}always-\/and-\/inside-\/groups\texorpdfstring{$\vert$}{|}never\mbox{]}$<$/tt$>$}{<tt>newlines-between: [ignore|always|always-and-inside-groups|never]</tt>}}\label{md_node__modules_2eslint-plugin-import_2docs_2rules_2order_autotoc_md11016}
Enforces or forbids new lines between import groups\+:


\begin{DoxyItemize}
\item If set to {\ttfamily ignore}, no errors related to new lines between import groups will be reported.
\item If set to {\ttfamily always}, at least one new line between each group will be enforced, and new lines inside a group will be forbidden. To prevent multiple lines between imports, core {\ttfamily no-\/multiple-\/empty-\/lines} rule can be used.
\item If set to {\ttfamily always-\/and-\/inside-\/groups}, it will act like {\ttfamily always} except newlines are allowed inside import groups.
\item If set to {\ttfamily never}, no new lines are allowed in the entire import section.
\end{DoxyItemize}

The default value is {\ttfamily "{}ignore"{}}.

Sort the order within each group in alphabetical manner based on {\bfseries{import path}}\+:


\begin{DoxyItemize}
\item {\ttfamily order}\+: use {\ttfamily asc} to sort in ascending order, and {\ttfamily desc} to sort in descending order (default\+: {\ttfamily ignore}).

\item {\ttfamily case\+Insensitive}\+: use {\ttfamily true} to ignore case, and {\ttfamily false} to consider case (default\+: {\ttfamily false}).
\end{DoxyItemize}
