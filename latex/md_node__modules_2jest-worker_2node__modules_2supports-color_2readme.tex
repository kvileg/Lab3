\hypertarget{md_node__modules_2jest-worker_2node__modules_2supports-color_2readme_autotoc_md18016}{}\doxysection{\texorpdfstring{API}{API}}\label{md_node__modules_2jest-worker_2node__modules_2supports-color_2readme_autotoc_md18016}
Returns an {\ttfamily Object} with a {\ttfamily stdout} and {\ttfamily stderr} property for testing either streams. Each property is an {\ttfamily Object}, or {\ttfamily false} if color is not supported.

The {\ttfamily stdout}/{\ttfamily stderr} objects specifies a level of support for color through a {\ttfamily .level} property and a corresponding flag\+:


\begin{DoxyItemize}
\item {\ttfamily .level = 1} and {\ttfamily .has\+Basic = true}\+: Basic color support (16 colors)
\item {\ttfamily .level = 2} and {\ttfamily .has256 = true}\+: 256 color support
\item {\ttfamily .level = 3} and {\ttfamily .has16m = true}\+: Truecolor support (16 million colors)
\end{DoxyItemize}\hypertarget{md_node__modules_2jest-worker_2node__modules_2supports-color_2readme_autotoc_md18017}{}\doxysubsection{\texorpdfstring{‘require(\textquotesingle{}supports-\/color’).\+supports\+Color(stream, options?)\`{}}{‘require('supports-color’).supportsColor(stream, options?)`}}\label{md_node__modules_2jest-worker_2node__modules_2supports-color_2readme_autotoc_md18017}
Additionally, {\ttfamily supports-\/color} exposes the {\ttfamily .supports\+Color()} function that takes an arbitrary write stream (e.\+g. {\ttfamily process.\+stdout}) and an optional options object to (re-\/)evaluate color support for an arbitrary stream.

For example, `require(\textquotesingle{}supports-\/color').stdout{\ttfamily is the equivalent of}require(\textquotesingle{}supports-\/color\textquotesingle{}).supports\+Color(process.\+stdout)\`{}.

The options object supports a single boolean property {\ttfamily sniff\+Flags}. By default it is {\ttfamily true}, which instructs {\ttfamily supports\+Color()} to sniff {\ttfamily process.\+argv} for the multitude of {\ttfamily -\/-\/color} flags (see {\itshape Info} below). If {\ttfamily false}, then {\ttfamily process.\+argv} is not considered when determining color support.\hypertarget{md_node__modules_2jest-worker_2node__modules_2supports-color_2readme_autotoc_md18018}{}\doxysection{\texorpdfstring{Info}{Info}}\label{md_node__modules_2jest-worker_2node__modules_2supports-color_2readme_autotoc_md18018}
It obeys the {\ttfamily -\/-\/color} and {\ttfamily -\/-\/no-\/color} CLI flags.

For situations where using {\ttfamily -\/-\/color} is not possible, use the environment variable {\ttfamily FORCE\+\_\+\+COLOR=1} (level 1), {\ttfamily FORCE\+\_\+\+COLOR=2} (level 2), or {\ttfamily FORCE\+\_\+\+COLOR=3} (level 3) to forcefully enable color, or {\ttfamily FORCE\+\_\+\+COLOR=0} to forcefully disable. The use of {\ttfamily FORCE\+\_\+\+COLOR} overrides all other color support checks.

Explicit 256/\+Truecolor mode can be enabled using the {\ttfamily -\/-\/color=256} and {\ttfamily -\/-\/color=16m} flags, respectively.\hypertarget{md_node__modules_2jest-worker_2node__modules_2supports-color_2readme_autotoc_md18019}{}\doxysection{\texorpdfstring{Related}{Related}}\label{md_node__modules_2jest-worker_2node__modules_2supports-color_2readme_autotoc_md18019}
