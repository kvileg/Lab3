{\ttfamily common-\/tags} initially started out as two template tags I\textquotesingle{}d always find myself writing -\/ one for stripping indents, and one for trimming multiline strings down to a single line. In it\textquotesingle{}s prime, I was an avid user of \href{http://coffeescript.org}{\texttt{ Coffee\+Script}}, which had this behaviour by default as part of it\textquotesingle{}s block strings feature. I also started out programming in Ruby, which has a similar mechanism called Heredocs.

Over time, I found myself needing a few more template tags to cover edge cases -\/ ones that supported including arrays, or ones that helped to render out tiny bits of HTML not large enough to deserve their own file or an entire template engine. So I packaged all of these up into this module.

As more features were proposed, and I found myself needing a way to override the default settings to cover even more edge cases, I realized that my initial implementation wouldn\textquotesingle{}t be easy to scale.

So I re-\/wrote this module on top of a core architecture that makes use of transformer plugins which can be composed, imported independently and re-\/used.\hypertarget{md_node__modules_2common-tags_2readme_autotoc_md7918}{}\doxysection{\texorpdfstring{Why You Should Care}{Why You Should Care}}\label{md_node__modules_2common-tags_2readme_autotoc_md7918}
Tagged templates in ES2015 are a welcome feature. But, they have their downsides. One such downside is that they preserve all whitespace by default -\/ which makes multiline strings in source code look terrible.

Source code is not just for computers to interpret. Humans have to read it too üòÅ. If you care at all about how neat your source code is, or come from a \href{http://coffeescript.org/}{\texttt{ Coffee\+Script}} background and miss the \href{http://coffeescript.org/\#strings}{\texttt{ block string syntax}}, then you will love {\ttfamily common-\/tags}, as it was initially intended to bring this feature "{}back"{} to JS since it\textquotesingle{}s \href{https://github.com/declandewet/common-tags/commit/2595288d6c276439d98d1bcbbb0aa113f4f7cd86}{\texttt{ initial commit}}.

{\ttfamily common-\/tags} also exposes a means of composing pipelines of dynamic transformer plugins. As someone with a little experience writing tagged templates, I can admit that it is often the case that one tag might need to do the same thing as another tag before doing any further processing; for example -\/ a typical tag that renders out HTML could strip initial indents first, then worry about handling character escapes. Both steps could easily be useful as their own separate template tags, but there isn\textquotesingle{}t an immediately obvious way of composing the two together for maximum re-\/use. {\ttfamily common-\/tags} offers not one, but two ways of doing this.

Furthermore, I try to keep this project as transparently stable and updated as frequently as I possibly can. As you may have already seen by the project status table, {\ttfamily common-\/tags} is linted, well tested, tests are well covered, tests pass on both Unix and Windows operating systems, the popularity bandwidth is easily referenced and dependency health is in plain sight üòÑ. {\ttfamily common-\/tags} is also already used in production on a number of proprietary sites and dependent projects, and contributions are always welcome, as are \mbox{[}suggestions\mbox{]}(issues).\hypertarget{md_node__modules_2common-tags_2readme_autotoc_md7919}{}\doxysection{\texorpdfstring{See Who Is Using $<$tt$>$common-\/tags$<$/tt$>$}{See Who Is Using <tt>common-tags</tt>}}\label{md_node__modules_2common-tags_2readme_autotoc_md7919}
