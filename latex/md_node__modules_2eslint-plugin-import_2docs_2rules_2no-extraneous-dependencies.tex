Forbid the import of external modules that are not declared in the {\ttfamily package.\+json}\textquotesingle{}s {\ttfamily dependencies}, {\ttfamily dev\+Dependencies}, {\ttfamily optional\+Dependencies}, {\ttfamily peer\+Dependencies}, or {\ttfamily bundled\+Dependencies}. The closest parent {\ttfamily package.\+json} will be used. If no {\ttfamily package.\+json} is found, the rule will not lint anything. This behavior can be changed with the rule option {\ttfamily package\+Dir}. Normally ignores imports of modules marked internal, but this can be changed with the rule option {\ttfamily include\+Internal}. Type imports can be verified by specifying {\ttfamily include\+Types}.

Modules have to be installed for this rule to work.\hypertarget{md_node__modules_2eslint-plugin-import_2docs_2rules_2no-extraneous-dependencies_autotoc_md10923}{}\doxysection{\texorpdfstring{Options}{Options}}\label{md_node__modules_2eslint-plugin-import_2docs_2rules_2no-extraneous-dependencies_autotoc_md10923}
This rule supports the following options\+:

{\ttfamily dev\+Dependencies}\+: If set to {\ttfamily false}, then the rule will show an error when {\ttfamily dev\+Dependencies} are imported. Defaults to {\ttfamily true}. Type imports are ignored by default.

{\ttfamily optional\+Dependencies}\+: If set to {\ttfamily false}, then the rule will show an error when {\ttfamily optional\+Dependencies} are imported. Defaults to {\ttfamily true}.

{\ttfamily peer\+Dependencies}\+: If set to {\ttfamily false}, then the rule will show an error when {\ttfamily peer\+Dependencies} are imported. Defaults to {\ttfamily true}.

{\ttfamily bundled\+Dependencies}\+: If set to {\ttfamily false}, then the rule will show an error when {\ttfamily bundled\+Dependencies} are imported. Defaults to {\ttfamily true}.

You can set the options like this\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{"{}import/no-\/extraneous-\/dependencies"{}:\ ["{}error"{},\ \{"{}devDependencies"{}:\ false,\ "{}optionalDependencies"{}:\ false,\ "{}peerDependencies"{}:\ false\}]}

\end{DoxyCode}


You can also use an array of globs instead of literal booleans\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{"{}import/no-\/extraneous-\/dependencies"{}:\ ["{}error"{},\ \{"{}devDependencies"{}:\ ["{}**/*.test.js"{},\ "{}**/*.spec.js"{}]\}]}

\end{DoxyCode}


When using an array of globs, the setting will be set to {\ttfamily true} (no errors reported) if the name of the file being linted matches a single glob in the array, and {\ttfamily false} otherwise.

There are 2 boolean options to opt into checking extra imports that are normally ignored\+: {\ttfamily include\+Internal}, which enables the checking of internal modules, and {\ttfamily include\+Types}, which enables checking of type imports in Type\+Script.


\begin{DoxyCode}{0}
\DoxyCodeLine{"{}import/no-\/extraneous-\/dependencies"{}:\ ["{}error"{},\ \{"{}includeInternal"{}:\ true,\ "{}includeTypes"{}:\ true\}]}

\end{DoxyCode}


Also there is one more option called {\ttfamily package\+Dir}, this option is to specify the path to the folder containing package.\+json.

If provided as a relative path string, will be computed relative to the current working directory at linter execution time. If this is not ideal (does not work with some editor integrations), consider using {\ttfamily \+\_\+\+\_\+dirname} to provide a path relative to your configuration.


\begin{DoxyCode}{0}
\DoxyCodeLine{"{}import/no-\/extraneous-\/dependencies"{}:\ ["{}error"{},\ \{"{}packageDir"{}:\ './some-\/dir/'\}]}
\DoxyCodeLine{//\ or}
\DoxyCodeLine{"{}import/no-\/extraneous-\/dependencies"{}:\ ["{}error"{},\ \{"{}packageDir"{}:\ path.join(\_\_dirname,\ 'some-\/dir')\}]}

\end{DoxyCode}


It may also be an array of multiple paths, to support monorepos or other novel project folder layouts\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{"{}import/no-\/extraneous-\/dependencies"{}:\ ["{}error"{},\ \{"{}packageDir"{}:\ ['./some-\/dir/',\ './root-\/pkg']\}]}

\end{DoxyCode}
