The module name says it all, check if a protocol requires a given port.\hypertarget{md_node__modules_2requires-port_2README_autotoc_md23254}{}\doxysection{\texorpdfstring{Installation}{Installation}}\label{md_node__modules_2requires-port_2README_autotoc_md23254}
This module is intended to be used with browserify or Node.\+js and is distributed in the public npm registry. To install it simply run the following command from your CLI\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{npm\ install\ -\/-\/save\ requires-\/port}

\end{DoxyCode}
\hypertarget{md_node__modules_2requires-port_2README_autotoc_md23255}{}\doxysection{\texorpdfstring{Usage}{Usage}}\label{md_node__modules_2requires-port_2README_autotoc_md23255}
The module exports it self as function and requires 2 arguments\+:


\begin{DoxyEnumerate}
\item The port number, can be a string or number.
\item Protocol, can be {\ttfamily http}, {\ttfamily http\+:} or even {\ttfamily \href{https://yomoma.com}{\texttt{ https\+://yomoma.\+com}}}. We just split it at {\ttfamily \+:} and use the first result. We currently accept the following protocols\+:
\begin{DoxyItemize}
\item {\ttfamily http}
\item {\ttfamily https}
\item {\ttfamily ws}
\item {\ttfamily wss}
\item {\ttfamily ftp}
\item {\ttfamily gopher}
\item {\ttfamily file}
\end{DoxyItemize}
\end{DoxyEnumerate}

It returns a boolean that indicates if protocol requires this port to be added to your URL.


\begin{DoxyCode}{0}
\DoxyCodeLine{'use\ strict';}
\DoxyCodeLine{}
\DoxyCodeLine{var\ required\ =\ require('requires-\/port');}
\DoxyCodeLine{}
\DoxyCodeLine{console.log(required('8080',\ 'http'))\ //\ true}
\DoxyCodeLine{console.log(required('80',\ 'http'))\ \ \ //\ false}

\end{DoxyCode}
