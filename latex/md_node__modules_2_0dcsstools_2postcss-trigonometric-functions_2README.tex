Given the dynamic nature of custom properties it\textquotesingle{}s impossible to know what the variable value is, which means the plugin can\textquotesingle{}t compute a final value for the stylesheet.

Because of that, any usage that contains a {\ttfamily var} is skipped.\hypertarget{md_node__modules_2_0dcsstools_2postcss-trigonometric-functions_2README_autotoc_md771}{}\doxysection{\texorpdfstring{Units}{Units}}\label{md_node__modules_2_0dcsstools_2postcss-trigonometric-functions_2README_autotoc_md771}
\href{https://github.com/csstools/postcss-plugins/tree/main/plugins/postcss-trigonometric-functions}{\texttt{ Post\+CSS Trigonometric Functions}} lets you use different special units that are within the spec and computed at run time to be able to calculate the result of the trigonometric function.

The following units are supported\+:


\begin{DoxyItemize}
\item {\ttfamily pi}\+: Computes to {\ttfamily Math.\+PI} which is {\ttfamily 3.\+141592653589793}
\item {\ttfamily e}\+: Computes to {\ttfamily Math.\+E} which is {\ttfamily 2.\+718281828459045}
\item {\ttfamily infinity}, {\ttfamily -\/infinity}\+: Compute to {\ttfamily Infinity} and {\ttfamily -\/Infinity} respectively. Note that the usage is case insensitive so {\ttfamily In\+Fi\+Ni\+Ty} is a valid value.
\end{DoxyItemize}

Some calculations (such as {\ttfamily sin(-\/infinity)}) might return {\ttfamily NaN} as per the spec. Given that {\ttfamily NaN} can\textquotesingle{}t be replaced with a value that\textquotesingle{}s useful to CSS it is left as is, as the result will be effectively ignored by the browser.\hypertarget{md_node__modules_2_0dcsstools_2postcss-trigonometric-functions_2README_autotoc_md772}{}\doxysection{\texorpdfstring{Options}{Options}}\label{md_node__modules_2_0dcsstools_2postcss-trigonometric-functions_2README_autotoc_md772}
