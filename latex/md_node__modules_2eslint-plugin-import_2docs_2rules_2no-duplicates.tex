Reports if a resolved path is imported more than once.

ESLint core has a similar rule (\href{https://eslint.org/docs/rules/no-duplicate-imports}{\texttt{ {\ttfamily no-\/duplicate-\/imports}}}), but this version is different in two key ways\+:


\begin{DoxyEnumerate}
\item the paths in the source code don\textquotesingle{}t have to exactly match, they just have to point to the same module on the filesystem. (i.\+e. {\ttfamily ./foo} and {\ttfamily ./foo.js})
\item this version distinguishes Flow {\ttfamily type} imports from standard imports. (\href{https://github.com/import-js/eslint-plugin-import/pull/334}{\texttt{ \#334}})
\end{DoxyEnumerate}\hypertarget{md_node__modules_2eslint-plugin-import_2docs_2rules_2no-duplicates_autotoc_md10910}{}\doxysection{\texorpdfstring{Rule Details}{Rule Details}}\label{md_node__modules_2eslint-plugin-import_2docs_2rules_2no-duplicates_autotoc_md10910}
Valid\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{import\ SomeDefaultClass,\ *\ as\ names\ from\ './mod'}
\DoxyCodeLine{//\ Flow\ \`{}type`\ import\ from\ same\ module\ is\ fine}
\DoxyCodeLine{import\ type\ SomeType\ from\ './mod'}

\end{DoxyCode}


...whereas here, both {\ttfamily ./mod} imports will be reported\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{import\ SomeDefaultClass\ from\ './mod'}
\DoxyCodeLine{}
\DoxyCodeLine{//\ oops,\ some\ other\ import\ separated\ these\ lines}
\DoxyCodeLine{import\ foo\ from\ './some-\/other-\/mod'}
\DoxyCodeLine{}
\DoxyCodeLine{import\ *\ as\ names\ from\ './mod'}
\DoxyCodeLine{}
\DoxyCodeLine{//\ will\ catch\ this\ too,\ assuming\ it\ is\ the\ same\ target\ module}
\DoxyCodeLine{import\ \{\ something\ \}\ from\ './mod.js'}

\end{DoxyCode}


The motivation is that this is likely a result of two developers importing different names from the same module at different times (and potentially largely different locations in the file.) This rule brings both (or n-\/many) to attention.\hypertarget{md_node__modules_2eslint-plugin-import_2docs_2rules_2no-duplicates_autotoc_md10911}{}\doxysubsection{\texorpdfstring{Query Strings}{Query Strings}}\label{md_node__modules_2eslint-plugin-import_2docs_2rules_2no-duplicates_autotoc_md10911}
By default, this rule ignores query strings (i.\+e. paths followed by a question mark), and thus imports from {\ttfamily ./mod?a} and {\ttfamily ./mod?b} will be considered as duplicates. However you can use the option {\ttfamily consider\+Query\+String} to handle them as different (primarily because browsers will resolve those imports differently).

Config\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{"{}import/no-\/duplicates"{}:\ ["{}error"{},\ \{"{}considerQueryString"{}:\ true\}]}

\end{DoxyCode}


And then the following code becomes valid\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{import\ minifiedMod\ from\ './mod?minify'}
\DoxyCodeLine{import\ noCommentsMod\ from\ './mod?comments=0'}
\DoxyCodeLine{import\ originalMod\ from\ './mod'}

\end{DoxyCode}


It will still catch duplicates when using the same module and the exact same query string\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{import\ SomeDefaultClass\ from\ './mod?minify'}
\DoxyCodeLine{}
\DoxyCodeLine{//\ This\ is\ invalid,\ assuming\ \`{}./mod`\ and\ \`{}./mod.js`\ are\ the\ same\ target:}
\DoxyCodeLine{import\ *\ from\ './mod.js?minify'}

\end{DoxyCode}
\hypertarget{md_node__modules_2eslint-plugin-import_2docs_2rules_2no-duplicates_autotoc_md10912}{}\doxysubsection{\texorpdfstring{Inline Type imports}{Inline Type imports}}\label{md_node__modules_2eslint-plugin-import_2docs_2rules_2no-duplicates_autotoc_md10912}
Type\+Script 4.\+5 introduced a new \href{https://devblogs.microsoft.com/typescript/announcing-typescript-4-5/\#type-on-import-names}{\texttt{ feature}} that allows mixing of named value and type imports. In order to support fixing to an inline type import when duplicate imports are detected, {\ttfamily prefer-\/inline} can be set to true.

Config\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{"{}import/no-\/duplicates"{}:\ ["{}error"{},\ \{"{}prefer-\/inline"{}:\ true\}]}

\end{DoxyCode}

If the core ESLint version is good enough (i.\+e. you\textquotesingle{}re {\itshape not} using Flow and you {\itshape are} using \doxysectlink{md_node__modules_2eslint-plugin-import_2docs_2rules_2extensions}{\`{}import/extensions\`{}}{0}), keep it and don\textquotesingle{}t use this.

If you like to split up imports across lines or may need to import a default and a namespace, you may not want to enable this rule.
