Inside the {\ttfamily enter} function, calling {\ttfamily this.\+skip()} will prevent the node\textquotesingle{}s children being walked, or the {\ttfamily leave} function (which is optional) being called.

Call {\ttfamily this.\+replace(new\+\_\+node)} in either {\ttfamily enter} or {\ttfamily leave} to replace the current node with a new one.

Call {\ttfamily this.\+remove()} in either {\ttfamily enter} or {\ttfamily leave} to remove the current node.\hypertarget{md_node__modules_2estree-walker_2README_autotoc_md12606}{}\doxysection{\texorpdfstring{Why not use estraverse?}{Why not use estraverse?}}\label{md_node__modules_2estree-walker_2README_autotoc_md12606}
The ESTree spec is evolving to accommodate ES6/7. I\textquotesingle{}ve had a couple of experiences where \href{https://github.com/estools/estraverse}{\texttt{ estraverse}} was unable to handle an AST generated by recent versions of acorn, because it hard-\/codes visitor keys.

estree-\/walker, by contrast, simply enumerates a node\textquotesingle{}s properties to find child nodes (and child lists of nodes), and is therefore resistant to spec changes. It\textquotesingle{}s also much smaller. (The performance, if you\textquotesingle{}re wondering, is basically identical.)

None of which should be taken as criticism of estraverse, which has more features and has been battle-\/tested in many more situations, and for which I\textquotesingle{}m very grateful.\hypertarget{md_node__modules_2estree-walker_2README_autotoc_md12607}{}\doxysection{\texorpdfstring{License}{License}}\label{md_node__modules_2estree-walker_2README_autotoc_md12607}
