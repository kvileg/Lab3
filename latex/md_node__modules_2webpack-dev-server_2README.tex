Use \href{https://webpack.js.org}{\texttt{ webpack}} with a development server that provides live reloading. This should be used for {\bfseries{development only}}.

It uses \href{https://github.com/webpack/webpack-dev-middleware}{\texttt{ webpack-\/dev-\/middleware}} under the hood, which provides fast in-\/memory access to the webpack assets.\hypertarget{md_node__modules_2webpack-dev-server_2README_autotoc_md29559}{}\doxysubsection{\texorpdfstring{Table of Contents}{Table of Contents}}\label{md_node__modules_2webpack-dev-server_2README_autotoc_md29559}

First things first, install the module\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{npm\ install\ webpack-\/dev-\/server\ -\/-\/save-\/dev}

\end{DoxyCode}


or


\begin{DoxyCode}{0}
\DoxyCodeLine{yarn\ add\ -\/D\ webpack-\/dev-\/server}

\end{DoxyCode}


or


\begin{DoxyCode}{0}
\DoxyCodeLine{pnpm\ add\ -\/D\ webpack-\/dev-\/server}

\end{DoxyCode}


{\itshape Note\+: While you can install and run webpack-\/dev-\/server globally, we recommend installing it locally. webpack-\/dev-\/server will always use a local installation over a global one.}\hypertarget{md_node__modules_2webpack-dev-server_2README_autotoc_md29561}{}\doxysubsection{\texorpdfstring{Usage}{Usage}}\label{md_node__modules_2webpack-dev-server_2README_autotoc_md29561}
There are two main, recommended methods of using the module\+:\hypertarget{md_node__modules_2webpack-dev-server_2README_autotoc_md29562}{}\doxysubsubsection{\texorpdfstring{With the CLI}{With the CLI}}\label{md_node__modules_2webpack-dev-server_2README_autotoc_md29562}
The easiest way to use it is with the \href{https://webpack.js.org/api/cli/}{\texttt{ webpack CLI}}. In the directory where your {\ttfamily webpack.\+config.\+js} is, run\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{npx\ webpack\ serve}

\end{DoxyCode}

Either method will start a server instance and begin listening for connections from {\ttfamily localhost} on port {\ttfamily 8080}.

webpack-\/dev-\/server is configured by default to support live-\/reload of files as you edit your assets while the server is running.

See \href{https://webpack.js.org/configuration/dev-server/\#devserver}{\texttt{ {\bfseries{the documentation}}}} for more use cases and options.\hypertarget{md_node__modules_2webpack-dev-server_2README_autotoc_md29567}{}\doxysubsection{\texorpdfstring{Browser Support}{Browser Support}}\label{md_node__modules_2webpack-dev-server_2README_autotoc_md29567}
While {\ttfamily webpack-\/dev-\/server} transpiles the client (browser) scripts to an ES5 state, the project only officially supports the {\itshape last two versions of major browsers}. We simply don\textquotesingle{}t have the resources to support every whacky browser out there.
