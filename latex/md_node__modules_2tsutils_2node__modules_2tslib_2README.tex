 This is a runtime library for \href{http://www.typescriptlang.org/}{\texttt{ Type\+Script}} that contains all of the Type\+Script helper functions.

This library is primarily used by the {\ttfamily -\/-\/import\+Helpers} flag in Type\+Script. When using {\ttfamily -\/-\/import\+Helpers}, a module that uses helper functions like {\ttfamily \+\_\+\+\_\+extends} and {\ttfamily \+\_\+\+\_\+assign} in the following emitted file\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{var\ \_\_assign\ =\ (this\ \&\&\ this.\_\_assign)\ ||\ Object.assign\ ||\ function(t)\ \{}
\DoxyCodeLine{\ \ \ \ for\ (var\ s,\ i\ =\ 1,\ n\ =\ arguments.length;\ i\ <\ n;\ i++)\ \{}
\DoxyCodeLine{\ \ \ \ \ \ \ \ s\ =\ arguments[i];}
\DoxyCodeLine{\ \ \ \ \ \ \ \ for\ (var\ p\ in\ s)\ if\ (Object.prototype.hasOwnProperty.call(s,\ p))}
\DoxyCodeLine{\ \ \ \ \ \ \ \ \ \ \ \ t[p]\ =\ s[p];}
\DoxyCodeLine{\ \ \ \ \}}
\DoxyCodeLine{\ \ \ \ return\ t;}
\DoxyCodeLine{\};}
\DoxyCodeLine{exports.x\ =\ \{\};}
\DoxyCodeLine{exports.y\ =\ \_\_assign(\{\},\ exports.x);}

\end{DoxyCode}


will instead be emitted as something like the following\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{var\ tslib\_1\ =\ require("{}tslib"{});}
\DoxyCodeLine{exports.x\ =\ \{\};}
\DoxyCodeLine{exports.y\ =\ tslib\_1.\_\_assign(\{\},\ exports.x);}

\end{DoxyCode}


Because this can avoid duplicate declarations of things like {\ttfamily \+\_\+\+\_\+extends}, {\ttfamily \+\_\+\+\_\+assign}, etc., this means delivering users smaller files on average, as well as less runtime overhead.
