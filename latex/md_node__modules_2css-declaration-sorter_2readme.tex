\href{https://www.npmjs.com/package/css-declaration-sorter}{\texttt{ }}

A Node.\+js module and \href{https://github.com/postcss/postcss}{\texttt{ Post\+CSS}} plugin to sort CSS, SCSS or Less declarations based on their property names. Ensuring styling is organized, more consistent and in order... The goal of this package is to sort the source code of a project in the build process or to decrease the distributed CSS gzipped size.

Check out \href{https://github.com/Siilwyn/prettier-plugin-css-order}{\texttt{ the Prettier plugin}} for usage with a variety of file formats.\hypertarget{md_node__modules_2css-declaration-sorter_2readme_autotoc_md8393}{}\doxysubsection{\texorpdfstring{Niceness}{Niceness}}\label{md_node__modules_2css-declaration-sorter_2readme_autotoc_md8393}

\begin{DoxyItemize}
\item Up-\/to-\/date CSS properties fetched from the \href{https://github.com/mdn/browser-compat-data/}{\texttt{ MDN Compatibility Data}} project.
\item Choose your wanted order or provide your own.
\item Nested rules sorting support.
\item SCSS and Less support when combined with either \href{https://github.com/postcss/postcss-scss}{\texttt{ postcss-\/scss}} or \href{https://github.com/webschik/postcss-less}{\texttt{ postcss-\/less}}.
\item Thought-\/out sorting orders out of the box, {\bfseries{approved by their authors}}.

\hypertarget{md_node__modules_2css-declaration-sorter_2readme_autotoc_md8401}{}\doxyparagraph{\texorpdfstring{order}{order}}\label{md_node__modules_2css-declaration-sorter_2readme_autotoc_md8401}
Type\+: {\ttfamily string} or {\ttfamily function} ~\newline
 Default\+: {\ttfamily alphabetical} ~\newline
 Options\+: {\ttfamily alphabetical}, {\ttfamily smacss}, {\ttfamily concentric-\/css}

Provide the name of one of the built-\/in sort orders or a comparison function that is passed to (\href{https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Array/sort}{\texttt{ {\ttfamily Array.\+sort}}}). This function receives two declaration names and is expected to return {\ttfamily -\/1}, {\ttfamily 0} or {\ttfamily 1} depending on the wanted order.\hypertarget{md_node__modules_2css-declaration-sorter_2readme_autotoc_md8402}{}\doxyparagraph{\texorpdfstring{keep\+Overrides}{keepOverrides}}\label{md_node__modules_2css-declaration-sorter_2readme_autotoc_md8402}
Type\+: {\ttfamily Boolean} ~\newline
 Default\+: {\ttfamily false} ~\newline


To prevent breaking legacy CSS where shorthand declarations override longhand declarations (also taking into account vendor prefixes) this option can enabled. For example {\ttfamily animation-\/name\+: some; animation\+: greeting;} will be kept in this order when {\ttfamily keep\+Overrides} is {\ttfamily true}.
