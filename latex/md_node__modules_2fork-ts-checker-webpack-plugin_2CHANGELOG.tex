This is the first major version of {\ttfamily fork-\/ts-\/checker-\/webpack-\/plugin}. A long time coming \+:-\/)

There are actually no breaking changes that we\textquotesingle{}re aware of; users of 0.\+x {\ttfamily fork-\/ts-\/checker-\/webpack-\/plugin} should be be able to upgrade without any drama. Users of Type\+Script 3+ may notice a performance improvement as by default the plugin now uses the \href{https://github.com/Microsoft/TypeScript/pull/20234}{\texttt{ incremental watch API}} in Type\+Script. Should this prove problematic you can opt out of using it by supplying {\ttfamily use\+Typescript\+Incremental\+Api\+: false}.

We are aware of an \href{https://github.com/TypeStrong/fork-ts-checker-webpack-plugin/issues/219}{\texttt{ issue with Vue and the incremental API}}. We hope it will be fixed soon -\/ a generous member of the community is taking a look. In the meantime, we will {\itshape not} default to using the incremental watch API when in Vue mode.

The plugin supports webpack 2, 3, 4 and 5 alpha and Type\+Script 2.\+1+ alongside tslint 4+.
