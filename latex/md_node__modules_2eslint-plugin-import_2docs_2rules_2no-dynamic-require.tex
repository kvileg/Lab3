The {\ttfamily require} method from Common\+JS is used to import modules from different files. Unlike the ES6 {\ttfamily import} syntax, it can be given expressions that will be resolved at runtime. While this is sometimes necessary and useful, in most cases it isn\textquotesingle{}t. Using expressions (for instance, concatenating a path and variable) as the argument makes it harder for tools to do static code analysis, or to find where in the codebase a module is used.

This rule forbids every call to {\ttfamily require()} that uses expressions for the module name argument.\hypertarget{md_node__modules_2eslint-plugin-import_2docs_2rules_2no-dynamic-require_autotoc_md10915}{}\doxysection{\texorpdfstring{Rule Details}{Rule Details}}\label{md_node__modules_2eslint-plugin-import_2docs_2rules_2no-dynamic-require_autotoc_md10915}
