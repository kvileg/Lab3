If a default import is requested, this rule will report if there is no default export in the imported module.

For \href{https://github.com/leebyron/ecmascript-more-export-from}{\texttt{ ES7}}, reports if a default is named and exported but is not found in the referenced module.

Note\+: for packages, the plugin will find exported names from \href{https://github.com/rollup/rollup/wiki/jsnext:main}{\texttt{ {\ttfamily jsnext\+:main}}}, if present in {\ttfamily package.\+json}. Redux\textquotesingle{}s npm module includes this key, and thereby is lintable, for example.

A module path that is \href{../README.md\#importignore}{\texttt{ ignored}} or not \href{https://github.com/bmeck/UnambiguousJavaScriptGrammar}{\texttt{ unambiguously an ES module}} will not be reported when imported.\hypertarget{md_node__modules_2eslint-plugin-import_2docs_2rules_2default_autotoc_md10817}{}\doxysection{\texorpdfstring{Rule Details}{Rule Details}}\label{md_node__modules_2eslint-plugin-import_2docs_2rules_2default_autotoc_md10817}

If you are using Common\+JS and/or modifying the exported namespace of any module at runtime, you will likely see false positives with this rule.

This rule currently does not interpret {\ttfamily module.\+exports = ...} as a {\ttfamily default} export, either, so such a situation will be reported in the importing module.\hypertarget{md_node__modules_2eslint-plugin-import_2docs_2rules_2default_autotoc_md10819}{}\doxysection{\texorpdfstring{Further Reading}{Further Reading}}\label{md_node__modules_2eslint-plugin-import_2docs_2rules_2default_autotoc_md10819}
