If you are new to {\ttfamily eslint}, below are a few resources that will help you to familiarize yourself with the project.


\begin{DoxyItemize}

If you would like to implement something, firstly\+: thanks! Community contributions are a magical thing. Like Redux or \href{https://youtu.be/SR5BfQ4rEqQ?t=2m25s}{\texttt{ the flux capacitor}}, they make open source possible.

{\bfseries{Working on your first Pull Request?}} You can learn how from this {\itshape free} series \href{https://egghead.io/series/how-to-contribute-to-an-open-source-project-on-github}{\texttt{ How to Contribute to an Open Source Project on Git\+Hub}}.

Here are some things to keep in mind when working on a PR\+:

{\bfseries{Trying to update an inactive Pull Request?}} If a PR is open, but unfortunately the author is, for any reason, not available to apply code review fixes or rebase the source branch, then please {\bfseries{do not open a new PR}}. Instead, paste a link to your own branch in the PR, and the maintainers can pull in your changes and update the existing PR in-\/place.\hypertarget{md_node__modules_2eslint-plugin-import_2CONTRIBUTING_autotoc_md10805}{}\doxysubsection{\texorpdfstring{Tests}{Tests}}\label{md_node__modules_2eslint-plugin-import_2CONTRIBUTING_autotoc_md10805}
A PR that is just failing test cases for an existing issue is very helpful, as this can take as much time (if not more) as it takes to implement a new feature or fix a bug.

If you only have enough time to write tests, fantastic! Submit away. This is a great jumping-\/off point for a core contributor or even another PR to continue what you\textquotesingle{}ve started.\hypertarget{md_node__modules_2eslint-plugin-import_2CONTRIBUTING_autotoc_md10806}{}\doxysubsection{\texorpdfstring{Docs}{Docs}}\label{md_node__modules_2eslint-plugin-import_2CONTRIBUTING_autotoc_md10806}
For enhancements to rules, please update the docs in {\ttfamily docs/rules} matching the rule filename from {\ttfamily src/rules} or the rule description in {\ttfamily meta.\+docs.\+description}. Running {\ttfamily npm run update\+:eslint-\/docs} will update the \doxysectlink{md_README}{README.md}{0} and rule doc header.

Bugfixes may not warrant docs changes, though it\textquotesingle{}s worth skimming the existing docs to see if there are any relevant caveats that need to be removed.\hypertarget{md_node__modules_2eslint-plugin-import_2CONTRIBUTING_autotoc_md10807}{}\doxysubsection{\texorpdfstring{Changelog}{Changelog}}\label{md_node__modules_2eslint-plugin-import_2CONTRIBUTING_autotoc_md10807}
Please add a quick blurb to the \href{./CHANGELOG.md\#unreleased}{\texttt{ {\bfseries{Unreleased}}}} section of the change log. Give yourself some credit, and please link back to the PR for future reference. This is especially helpful for resolver changes, as the resolvers are less frequently modified and published.

Note also that the change log can\textquotesingle{}t magically link back to Github entities (i.\+e. PRs, issues, users) or rules; there are a handful of footnote URL definitions at the bottom.
