Example JSONPath expressions\+:

\tabulinesep=1mm
\begin{longtabu}spread 0pt [c]{*{2}{|X[-1]}|}
\hline
\PBS\centering \cellcolor{\tableheadbgcolor}\textbf{ JSONPath   }&\PBS\centering \cellcolor{\tableheadbgcolor}\textbf{ Description    }\\\cline{1-2}
\endfirsthead
\hline
\endfoot
\hline
\PBS\centering \cellcolor{\tableheadbgcolor}\textbf{ JSONPath   }&\PBS\centering \cellcolor{\tableheadbgcolor}\textbf{ Description    }\\\cline{1-2}
\endhead
{\ttfamily \$.store.\+book\mbox{[}\texorpdfstring{$\ast$}{*}\mbox{]}.author}   &The authors of all books in the store    \\\cline{1-2}
{\ttfamily \$..author}   &All authors    \\\cline{1-2}
{\ttfamily \$.store.\texorpdfstring{$\ast$}{*}}   &All things in store, which are some books and a red bicycle    \\\cline{1-2}
{\ttfamily \$.store..price}   &The price of everything in the store    \\\cline{1-2}
{\ttfamily \$..book\mbox{[}2\mbox{]}}   &The third book    \\\cline{1-2}
{\ttfamily \$..book\mbox{[}(@.length-\/1)\mbox{]}}   &The last book via script subscript    \\\cline{1-2}
{\ttfamily \$..book\mbox{[}-\/1\+:\mbox{]}}   &The last book via slice    \\\cline{1-2}
{\ttfamily \$..book\mbox{[}0,1\mbox{]}}   &The first two books via subscript union    \\\cline{1-2}
{\ttfamily \$..book\mbox{[}\+:2\mbox{]}}   &The first two books via subscript array slice    \\\cline{1-2}
{\ttfamily \$..book\mbox{[}?(@.isbn)\mbox{]}}   &Filter all books with isbn number    \\\cline{1-2}
{\ttfamily \$..book\mbox{[}?(@.price\texorpdfstring{$<$}{<}10)\mbox{]}}   &Filter all books cheaper than 10    \\\cline{1-2}
{\ttfamily \$..book\mbox{[}?(@.price==8.\+95)\mbox{]}}   &Filter all books that cost 8.\+95    \\\cline{1-2}
{\ttfamily \$..book\mbox{[}?(@.price\texorpdfstring{$<$}{<}30 \&\& @.category=="{}fiction"{})\mbox{]}}   &Filter all fiction books cheaper than 30    \\\cline{1-2}
{\ttfamily \$..\texorpdfstring{$\ast$}{*}}   &All members of JSON structure   \\\cline{1-2}
\end{longtabu}

\hypertarget{md_node__modules_2jsonpath_2README_autotoc_md18455}{}\doxysubsection{\texorpdfstring{Differences from Original Implementation}{Differences from Original Implementation}}\label{md_node__modules_2jsonpath_2README_autotoc_md18455}
This implementation aims to be compatible with Stefan Goessner\textquotesingle{}s original implementation with a few notable exceptions described below.\hypertarget{md_node__modules_2jsonpath_2README_autotoc_md18456}{}\doxyparagraph{\texorpdfstring{Evaluating Script Expressions}{Evaluating Script Expressions}}\label{md_node__modules_2jsonpath_2README_autotoc_md18456}
Script expressions (i.\+e, {\ttfamily (...)} and {\ttfamily ?(...)}) are statically evaluated via \href{https://github.com/substack/static-eval}{\texttt{ static-\/eval}} rather than using the underlying script engine directly. That means both that the scope is limited to the instance variable ({\ttfamily @}), and only simple expressions (with no side effects) will be valid. So for example, {\ttfamily ?(@.length\texorpdfstring{$>$}{>}10)} will be just fine to match arrays with more than ten elements, but {\ttfamily ?(process.\+exit())} will not get evaluated since {\ttfamily process} would yield a {\ttfamily Reference\+Error}. This method is even safer than {\ttfamily vm.\+run\+In\+New\+Context}, since the script engine itself is more limited and entirely distinct from the one running the application code. See more details in the \href{https://github.com/substack/static-eval/blob/master/index.js}{\texttt{ implementation}} of the evaluator.\hypertarget{md_node__modules_2jsonpath_2README_autotoc_md18457}{}\doxyparagraph{\texorpdfstring{Grammar}{Grammar}}\label{md_node__modules_2jsonpath_2README_autotoc_md18457}
This project uses a formal BNF \href{https://github.com/dchester/jsonpath/blob/master/lib/grammar.js}{\texttt{ grammar}} to parse JSONPath expressions, an attempt at reverse-\/engineering the intent of the original implementation, which parses via a series of creative regular expressions. The original regex approach can sometimes be forgiving for better or for worse (e.\+g., {\ttfamily \$\mbox{[}\textquotesingle{}store\mbox{]}} =\texorpdfstring{$>$}{>} `\$\mbox{[}\textquotesingle{}store'\mbox{]}{\ttfamily ), and in other cases, can be just plain wrong (e.\+g.}\mbox{[}{\ttfamily =\texorpdfstring{$>$}{>}}\$\`{}).\hypertarget{md_node__modules_2jsonpath_2README_autotoc_md18458}{}\doxyparagraph{\texorpdfstring{Other Minor Differences}{Other Minor Differences}}\label{md_node__modules_2jsonpath_2README_autotoc_md18458}
As a result of using a real parser and static evaluation, there are some arguable bugs in the original library that have not been carried through here\+:


\begin{DoxyItemize}
\item strings in subscripts may now be double-\/quoted
\item final {\ttfamily step} arguments in slice operators may now be negative
\item script expressions may now contain {\ttfamily .} and {\ttfamily @} characters not referring to instance variables
\item subscripts no longer act as character slices on string elements
\item non-\/ascii non-\/word characters are no-\/longer valid in member identifier names; use quoted subscript strings instead (e.\+g., `\$\mbox{[}'\$\textquotesingle{}\mbox{]}{\ttfamily instead of}\$.\$\`{})
\item unions now yield real unions with no duplicates rather than concatenated results
