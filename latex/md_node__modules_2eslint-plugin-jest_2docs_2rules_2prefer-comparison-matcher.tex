 Jest has a number of built-\/in matchers for comparing numbers which allow for more readable tests and error messages if an expectation fails.\hypertarget{md_node__modules_2eslint-plugin-jest_2docs_2rules_2prefer-comparison-matcher_autotoc_md11438}{}\doxysection{\texorpdfstring{Rule details}{Rule details}}\label{md_node__modules_2eslint-plugin-jest_2docs_2rules_2prefer-comparison-matcher_autotoc_md11438}
This rule checks for comparisons in tests that could be replaced with one of the following built-\/in comparison matchers\+:


\begin{DoxyItemize}
\item {\ttfamily to\+Be\+Greater\+Than}
\item {\ttfamily to\+Be\+Greater\+Than\+Or\+Equal}
\item {\ttfamily to\+Be\+Less\+Than}
\item {\ttfamily to\+Be\+Less\+Than\+Or\+Equal}
\end{DoxyItemize}

Examples of {\bfseries{incorrect}} code for this rule\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{expect(x\ >\ 5).toBe(true);}
\DoxyCodeLine{expect(x\ <\ 7).not.toEqual(true);}
\DoxyCodeLine{expect(x\ <=\ y).toStrictEqual(true);}

\end{DoxyCode}


Examples of {\bfseries{correct}} code for this rule\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{expect(x).toBeGreaterThan(5);}
\DoxyCodeLine{expect(x).not.toBeLessThanOrEqual(7);}
\DoxyCodeLine{expect(x).toBeLessThanOrEqual(y);}
\DoxyCodeLine{}
\DoxyCodeLine{//\ special\ case\ -\/\ see\ below}
\DoxyCodeLine{expect(x\ <\ 'Carl').toBe(true);}

\end{DoxyCode}


Note that these matchers only work with numbers and bigints, and that the rule assumes that any variables on either side of the comparison operator are of one of those types -\/ this means if you\textquotesingle{}re using the comparison operator with strings, the fix applied by this rule will result in an error.


\begin{DoxyCode}{0}
\DoxyCodeLine{expect(myName).toBeGreaterThanOrEqual(theirName);\ //\ Matcher\ error:\ received\ value\ must\ be\ a\ number\ or\ bigint}

\end{DoxyCode}


The reason for this is that comparing strings with these operators is expected to be very rare and would mean not being able to have an automatic fixer for this rule.

If for some reason you are using these operators to compare strings, you can disable this rule using an inline \href{https://eslint.org/docs/user-guide/configuring/rules\#disabling-rules}{\texttt{ configuration comment}}\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{//\ eslint-\/disable-\/next-\/line\ jest/prefer-\/comparison-\/matcher}
\DoxyCodeLine{expect(myName\ >\ theirName).toBe(true);}

\end{DoxyCode}
