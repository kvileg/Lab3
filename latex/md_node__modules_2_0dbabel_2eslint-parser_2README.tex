 {\bfseries{@babel/eslint-\/parser}} allows you to lint {\bfseries{ALL}} valid Babel code with the fantastic \href{https://github.com/eslint/eslint}{\texttt{ ESLint}}.\hypertarget{md_node__modules_2_0dbabel_2eslint-parser_2README_autotoc_md98}{}\doxysection{\texorpdfstring{When should I use @babel/eslint-\/parser?}{When should I use @babel/eslint-parser?}}\label{md_node__modules_2_0dbabel_2eslint-parser_2README_autotoc_md98}
ESLint\textquotesingle{}s default parser and core rules \href{https://github.com/eslint/eslint/blob/a675c89573836adaf108a932696b061946abf1e6/README.md\#what-about-experimental-features}{\texttt{ only support the latest final ECMAScript standard}} and do not support experimental (such as new features) and non-\/standard (such as Flow or Type\+Script types) syntax provided by Babel. @babel/eslint-\/parser is a parser that allows ESLint to run on source code that is transformed by Babel.

{\bfseries{Note\+:}} You only need to use @babel/eslint-\/parser if you are using Babel to transform your code. If this is not the case, please use the relevant parser for your chosen flavor of ECMAScript (note that the default parser supports all non-\/experimental syntax as well as JSX).\hypertarget{md_node__modules_2_0dbabel_2eslint-parser_2README_autotoc_md99}{}\doxysection{\texorpdfstring{How does it work?}{How does it work?}}\label{md_node__modules_2_0dbabel_2eslint-parser_2README_autotoc_md99}
ESLint allows for the use of \href{https://eslint.org/docs/developer-guide/working-with-custom-parsers}{\texttt{ custom parsers}}. When using this plugin, your code is parsed by Babel\textquotesingle{}s parser (using the configuration specified in your \href{https://babeljs.io/docs/en/configuration}{\texttt{ Babel configuration file}}) and the resulting AST is transformed into an \href{https://github.com/estree/estree}{\texttt{ ESTree}}-\/compliant structure that ESLint can understand. All location info such as line numbers, columns is also retained so you can track down errors with ease.

{\bfseries{Note\+:}} ESLint\textquotesingle{}s core rules do not support experimental syntax and may therefore not work as expected when using {\ttfamily @babel/eslint-\/parser}. Please use the companion \href{https://github.com/babel/babel/tree/main/eslint/babel-eslint-plugin}{\texttt{ {\ttfamily @babel/eslint-\/plugin}}} plugin for core rules that you have issues with.\hypertarget{md_node__modules_2_0dbabel_2eslint-parser_2README_autotoc_md100}{}\doxysection{\texorpdfstring{Usage}{Usage}}\label{md_node__modules_2_0dbabel_2eslint-parser_2README_autotoc_md100}

While \href{https://github.com/babel/babel/tree/main/eslint/babel-eslint-parser}{\texttt{ {\ttfamily @babel/eslint-\/parser}}} can parse Type\+Script, we don\textquotesingle{}t currently support linting Type\+Script using the rules in \href{https://github.com/babel/babel/tree/main/eslint/babel-eslint-plugin}{\texttt{ {\ttfamily @babel/eslint-\/plugin}}}. This is because the Type\+Script community has centered around \href{https://github.com/typescript-eslint/typescript-eslint}{\texttt{ {\ttfamily @typescript-\/eslint}}} and we want to avoid duplicate work. Additionally, since \href{https://github.com/typescript-eslint/typescript-eslint}{\texttt{ {\ttfamily @typescript-\/eslint}}} uses Type\+Script under the hood, its rules can be made type-\/aware, which is something Babel doesn\textquotesingle{}t have the ability to do.\hypertarget{md_node__modules_2_0dbabel_2eslint-parser_2README_autotoc_md106}{}\doxysection{\texorpdfstring{Questions and support}{Questions and support}}\label{md_node__modules_2_0dbabel_2eslint-parser_2README_autotoc_md106}
If you have an issue, please first check if it can be reproduced with the default parser and with the latest versions of {\ttfamily eslint} and {\ttfamily @babel/eslint-\/parser}. If it is not reproducible with the default parser, it is most likely an issue with {\ttfamily @babel/eslint-\/parser}.

For questions and support please visit the \href{https://babeljs.slack.com/messages/discussion/}{\texttt{ {\ttfamily \#discussion}}} Babel Slack channel (sign up \href{https://slack.babeljs.io/}{\texttt{ here}}) or the \href{https://eslint.org/chat}{\texttt{ ESLint Discord server}}.
