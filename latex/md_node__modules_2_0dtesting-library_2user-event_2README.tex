From \href{https://github.com/testing-library/dom-testing-library/issues/107}{\texttt{ testing-\/library/dom-\/testing-\/library\#107}}\+:

\begin{quote}
\mbox{[}...\mbox{]} it is becoming apparent the need to express user actions on a web page using a higher-\/level abstraction than \href{https://testing-library.com/docs/dom-testing-library/api-events\#fireevent}{\texttt{ {\ttfamily fire\+Event}}} \end{quote}
\hypertarget{md_node__modules_2_0dtesting-library_2user-event_2README_autotoc_md3182}{}\doxysubsection{\texorpdfstring{The solution}{The solution}}\label{md_node__modules_2_0dtesting-library_2user-event_2README_autotoc_md3182}
{\ttfamily user-\/event} tries to simulate the real events that would happen in the browser as the user interacts with it. For example {\ttfamily user\+Event.\+click(checkbox)} would change the state of the checkbox.

\begin{quote}
\href{https://twitter.com/kentcdodds/status/977018512689455106}{\texttt{ The more your tests resemble the way your software is used, the more confidence they can give you.}} \end{quote}
