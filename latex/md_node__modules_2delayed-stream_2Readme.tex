\hypertarget{md_node__modules_2delayed-stream_2Readme_autotoc_md9376}{}\doxysection{\texorpdfstring{Implementation}{Implementation}}\label{md_node__modules_2delayed-stream_2Readme_autotoc_md9376}
In order to use this meta stream properly, here are a few things you should know about the implementation.\hypertarget{md_node__modules_2delayed-stream_2Readme_autotoc_md9377}{}\doxysubsection{\texorpdfstring{Event Buffering / Proxying}{Event Buffering / Proxying}}\label{md_node__modules_2delayed-stream_2Readme_autotoc_md9377}
All events of the {\ttfamily source} stream are hijacked by overwriting the {\ttfamily source.\+emit} method. Until node implements a catch-\/all event listener, this is the only way.

However, delayed-\/stream still continues to emit all events it captures on the {\ttfamily source}, regardless of whether you have released the delayed stream yet or not.

Upon creation, delayed-\/stream captures all {\ttfamily source} events and stores them in an internal event buffer. Once {\ttfamily delayed\+Stream.\+release()} is called, all buffered events are emitted on the {\ttfamily delayed\+Stream}, and the event buffer is cleared. After that, delayed-\/stream merely acts as a proxy for the underlaying source.\hypertarget{md_node__modules_2delayed-stream_2Readme_autotoc_md9378}{}\doxysubsection{\texorpdfstring{Error handling}{Error handling}}\label{md_node__modules_2delayed-stream_2Readme_autotoc_md9378}
Error events on {\ttfamily source} are buffered / proxied just like any other events. However, {\ttfamily delayed\+Stream.\+create} attaches a no-\/op `\textquotesingle{}error'{\ttfamily listener to the }source{\ttfamily . This way you only have to handle errors on the}delayed\+Stream\`{} object, rather than in two places.\hypertarget{md_node__modules_2delayed-stream_2Readme_autotoc_md9379}{}\doxysubsection{\texorpdfstring{Buffer limits}{Buffer limits}}\label{md_node__modules_2delayed-stream_2Readme_autotoc_md9379}
delayed-\/stream provides a {\ttfamily max\+Data\+Size} property that can be used to limit the amount of data being buffered. In order to protect you from bad {\ttfamily source} streams that don\textquotesingle{}t react to {\ttfamily source.\+pause()}, this feature is enabled by default.\hypertarget{md_node__modules_2delayed-stream_2Readme_autotoc_md9380}{}\doxysection{\texorpdfstring{API}{API}}\label{md_node__modules_2delayed-stream_2Readme_autotoc_md9380}
