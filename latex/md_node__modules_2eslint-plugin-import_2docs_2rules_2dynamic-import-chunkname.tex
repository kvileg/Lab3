This rule reports any dynamic imports without a webpack\+Chunk\+Name specified in a leading block comment in the proper format.

This rule enforces naming of webpack chunks in dynamic imports. When you don\textquotesingle{}t explicitly name chunks, webpack will autogenerate chunk names that are not consistent across builds, which prevents long-\/term browser caching.\hypertarget{md_node__modules_2eslint-plugin-import_2docs_2rules_2dynamic-import-chunkname_autotoc_md10821}{}\doxysection{\texorpdfstring{Rule Details}{Rule Details}}\label{md_node__modules_2eslint-plugin-import_2docs_2rules_2dynamic-import-chunkname_autotoc_md10821}
This rule runs against {\ttfamily import()} by default, but can be configured to also run against an alternative dynamic-\/import function, e.\+g. \textquotesingle{}dynamic\+Import.\textquotesingle{} You can also configure the regex format you\textquotesingle{}d like to accept for the webpack\+Chunk\+Name -\/ for example, if we don\textquotesingle{}t want the number 6 to show up in our chunk names\+:
