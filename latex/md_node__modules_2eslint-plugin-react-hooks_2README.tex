 This ESLint plugin enforces the \href{https://reactjs.org/docs/hooks-rules.html}{\texttt{ Rules of Hooks}}.

It is a part of the \href{https://reactjs.org/docs/hooks-intro.html}{\texttt{ Hooks API}} for React.\hypertarget{md_node__modules_2eslint-plugin-react-hooks_2README_autotoc_md11894}{}\doxysection{\texorpdfstring{Installation}{Installation}}\label{md_node__modules_2eslint-plugin-react-hooks_2README_autotoc_md11894}
{\bfseries{Note\+: If you\textquotesingle{}re using Create React App, please use {\ttfamily react-\/scripts} \texorpdfstring{$>$}{>}= 3 instead of adding it directly.}}

\begin{DoxyCode}{0}
\DoxyCodeLine{\#\ npm}
\DoxyCodeLine{npm\ install\ eslint-\/plugin-\/react-\/hooks\ -\/-\/save-\/dev}
\DoxyCodeLine{}

\DoxyCodeLine{yarn\ add\ eslint-\/plugin-\/react-\/hooks\ -\/-\/dev}

{\ttfamily exhaustive-\/deps} can be configured to validate dependencies of custom Hooks with the {\ttfamily additional\+Hooks} option. This option accepts a regex to match the names of custom Hooks that have dependencies.


\begin{DoxyCode}{0}
\DoxyCodeLine{\{}
\DoxyCodeLine{\ \ "{}rules"{}:\ \{}
\DoxyCodeLine{\ \ \ \ //\ ...}
\DoxyCodeLine{\ \ \ \ "{}react-\/hooks/exhaustive-\/deps"{}:\ ["{}warn"{},\ \{}
\DoxyCodeLine{\ \ \ \ \ \ "{}additionalHooks"{}:\ "{}(useMyCustomHook|useMyOtherCustomHook)"{}}
\DoxyCodeLine{\ \ \ \ \}]}
\DoxyCodeLine{\ \ \}}
\DoxyCodeLine{\}}

\end{DoxyCode}


We suggest to use this option {\bfseries{very sparingly, if at all}}.
