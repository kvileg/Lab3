\hypertarget{md_node__modules_2eslint-plugin-testing-library_2README_autotoc_md11955}{}\doxysubsection{\texorpdfstring{Aggressive Reporting}{Aggressive Reporting}}\label{md_node__modules_2eslint-plugin-testing-library_2README_autotoc_md11955}
In v4 this plugin introduced a new feature called "{}\+Aggressive Reporting"{}, which intends to detect Testing Library utils usages even if they don\textquotesingle{}t come directly from a Testing Library package (i.\+e. \href{https://testing-library.com/docs/react-testing-library/setup/\#custom-render}{\texttt{ using a custom utility file to re-\/export everything from Testing Library}}). You can \href{docs/migration-guides/v4.md\#aggressive-reporting}{\texttt{ read more about this feature here}}.

If you are looking to restricting or switching off this feature, please refer to the Shared Settings section to do so.\hypertarget{md_node__modules_2eslint-plugin-testing-library_2README_autotoc_md11956}{}\doxysubsection{\texorpdfstring{Shared Settings}{Shared Settings}}\label{md_node__modules_2eslint-plugin-testing-library_2README_autotoc_md11956}
There are some configuration options available that will be shared across all the plugin rules. This is achieved using \href{https://eslint.org/docs/user-guide/configuring/configuration-files\#adding-shared-settings}{\texttt{ ESLint Shared Settings}}. These Shared Settings are meant to be used if you need to restrict or switch off the Aggressive Reporting, which is an out of the box advanced feature to lint Testing Library usages in a simpler way for most of the users. {\bfseries{So please before configuring any of these settings}}, read more about \href{docs/migration-guides/v4.md\#aggressive-reporting}{\texttt{ the advantages of {\ttfamily eslint-\/plugin-\/testing-\/library} Aggressive Reporting feature}}, and \href{docs/migration-guides/v4.md\#shared-settings}{\texttt{ how it\textquotesingle{}s affected by these settings}}.

If you find ESLint errors related to {\ttfamily eslint-\/plugin-\/testing-\/library} in files other than testing, this could be caused by Aggressive Reporting.

You can avoid this by\+:


\begin{DoxyEnumerate}
\item running \`{}eslint-\/plugin-\/testing-\/library\`{} only against testing files
\item limiting the scope of Aggressive Reporting through Shared Settings
\item switching Aggressive Reporting feature off
\end{DoxyEnumerate}

If you think the error you are getting is not related to this at all, please \href{https://github.com/testing-library/eslint-plugin-testing-library/issues/new/choose}{\texttt{ fill a new issue}} with as many details as possible.\hypertarget{md_node__modules_2eslint-plugin-testing-library_2README_autotoc_md11963}{}\doxysubsubsection{\texorpdfstring{False positives in testing files}{False positives in testing files}}\label{md_node__modules_2eslint-plugin-testing-library_2README_autotoc_md11963}
