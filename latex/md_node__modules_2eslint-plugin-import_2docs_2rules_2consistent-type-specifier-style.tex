In both Flow and Type\+Script you can mark an import as a type-\/only import by adding a "{}kind"{} marker to the import. Both languages support two positions for marker.

{\bfseries{At the top-\/level}} which marks all names in the import as type-\/only and applies to named, default, and namespace (for Type\+Script) specifiers\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{import\ type\ Foo\ from\ 'Foo';}
\DoxyCodeLine{import\ type\ \{Bar\}\ from\ 'Bar';}
\DoxyCodeLine{//\ ts\ only}
\DoxyCodeLine{import\ type\ *\ as\ Bam\ from\ 'Bam';}
\DoxyCodeLine{//\ flow\ only}
\DoxyCodeLine{import\ typeof\ Baz\ from\ 'Baz';}

\end{DoxyCode}


{\bfseries{Inline}} with to the named import, which marks just the specific name in the import as type-\/only. An inline specifier is only valid for named specifiers, and not for default or namespace specifiers\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{import\ \{type\ Foo\}\ from\ 'Foo';}
\DoxyCodeLine{//\ flow\ only}
\DoxyCodeLine{import\ \{typeof\ Bar\}\ from\ 'Bar';}

\end{DoxyCode}
\hypertarget{md_node__modules_2eslint-plugin-import_2docs_2rules_2consistent-type-specifier-style_autotoc_md10810}{}\doxysection{\texorpdfstring{Rule Details}{Rule Details}}\label{md_node__modules_2eslint-plugin-import_2docs_2rules_2consistent-type-specifier-style_autotoc_md10810}
This rule either enforces or bans the use of inline type-\/only markers for named imports.

This rule includes a fixer that will automatically convert your specifiers to the correct form -\/ however the fixer will not respect your preferences around de-\/duplicating imports. If this is important to you, consider using the \doxysectlink{md_node__modules_2eslint-plugin-import_2docs_2rules_2no-duplicates}{\`{}import/no-\/duplicates\`{}}{0} rule.\hypertarget{md_node__modules_2eslint-plugin-import_2docs_2rules_2consistent-type-specifier-style_autotoc_md10811}{}\doxysection{\texorpdfstring{Options}{Options}}\label{md_node__modules_2eslint-plugin-import_2docs_2rules_2consistent-type-specifier-style_autotoc_md10811}
The rule accepts a single string option which may be one of\+:


\begin{DoxyItemize}
\item `\textquotesingle{}prefer-\/inline'{\ttfamily -\/ enforces that named type-\/only specifiers are only ever written with an inline marker; and never as part of a top-\/level, type-\/only import. -\/}\textquotesingle{}prefer-\/top-\/level\textquotesingle{}\`{} -\/ enforces that named type-\/only specifiers only ever written as part of a top-\/level, type-\/only import; and never with an inline marker.
\end{DoxyItemize}

By default the rule will use the {\ttfamily prefer-\/inline} option.\hypertarget{md_node__modules_2eslint-plugin-import_2docs_2rules_2consistent-type-specifier-style_autotoc_md10812}{}\doxysection{\texorpdfstring{Examples}{Examples}}\label{md_node__modules_2eslint-plugin-import_2docs_2rules_2consistent-type-specifier-style_autotoc_md10812}
