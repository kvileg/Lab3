A somewhat JSON compatible interface for query string parsing. This query string parser is dumb, don\textquotesingle{}t expect to much from it as it only wants to parse simple query strings. If you want to parse complex, multi level and deeply nested query strings then you should ask your self. WTF am I doing?\hypertarget{md_node__modules_2querystringify_2README_autotoc_md22633}{}\doxysection{\texorpdfstring{Installation}{Installation}}\label{md_node__modules_2querystringify_2README_autotoc_md22633}
This module is released in npm as {\ttfamily querystringify}. It\textquotesingle{}s also compatible with {\ttfamily browserify} so it can be used on the server as well as on the client. To install it simply run the following command from your CLI\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{npm\ install\ -\/-\/save\ querystringify}

\end{DoxyCode}

\hypertarget{md_node__modules_2querystringify_2README_autotoc_md22635}{}\doxysubsection{\texorpdfstring{qs.\+parse()}{qs.parse()}}\label{md_node__modules_2querystringify_2README_autotoc_md22635}
The parse method transforms a given query string in to an object. Parameters without values are set to empty strings. It does not care if your query string is prefixed with a {\ttfamily ?}, a {\ttfamily \#}, or not prefixed. It just extracts the parts between the {\ttfamily =} and {\ttfamily \&}\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{qs.parse('?foo=bar');\ \ \ \ \ \ \ \ \ //\ \{\ foo:\ 'bar'\ \}}
\DoxyCodeLine{qs.parse('\#foo=bar');\ \ \ \ \ \ \ \ \ //\ \{\ foo:\ 'bar'\ \}}
\DoxyCodeLine{qs.parse('foo=bar');\ \ \ \ \ \ \ \ \ \ //\ \{\ foo:\ 'bar'\ \}}
\DoxyCodeLine{qs.parse('foo=bar\&bar=foo');\ \ //\ \{\ foo:\ 'bar',\ bar:\ 'foo'\ \}}
\DoxyCodeLine{qs.parse('foo\&bar=foo');\ \ \ \ \ \ //\ \{\ foo:\ '',\ bar:\ 'foo'\ \}}

\end{DoxyCode}
\hypertarget{md_node__modules_2querystringify_2README_autotoc_md22636}{}\doxysubsection{\texorpdfstring{qs.\+stringify()}{qs.stringify()}}\label{md_node__modules_2querystringify_2README_autotoc_md22636}
This transforms a given object in to a query string. By default we return the query string without a {\ttfamily ?} prefix. If you want to prefix it by default simply supply {\ttfamily true} as second argument. If it should be prefixed by something else simply supply a string with the prefix value as second argument\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{qs.stringify(\{\ foo:\ bar\ \});\ \ \ \ \ \ \ //\ foo=bar}
\DoxyCodeLine{qs.stringify(\{\ foo:\ bar\ \},\ true);\ //\ ?foo=bar}
\DoxyCodeLine{qs.stringify(\{\ foo:\ bar\ \},\ '\#');\ \ //\ \#foo=bar}
\DoxyCodeLine{qs.stringify(\{\ foo:\ ''\ \},\ '\&');\ \ \ //\ \&foo=}

\end{DoxyCode}
