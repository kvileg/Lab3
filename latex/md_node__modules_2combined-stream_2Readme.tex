 A stream that emits multiple other streams one after another.

{\bfseries{NB}} Currently {\ttfamily combined-\/stream} works with streams version 1 only. There is ongoing effort to switch this library to streams version 2. Any help is welcome. \+:) Meanwhile you can explore other libraries that provide streams2 support with more or less compatibility with {\ttfamily combined-\/stream}.

\hypertarget{md_node__modules_2combined-stream_2Readme_autotoc_md7859}{}\doxysection{\texorpdfstring{Usage}{Usage}}\label{md_node__modules_2combined-stream_2Readme_autotoc_md7859}
Here is a simple example that shows how you can use combined-\/stream to combine two files into one\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\ javascript}
\DoxyCodeLine{var\ CombinedStream\ =\ require('combined-\/stream');}
\DoxyCodeLine{var\ fs\ =\ require('fs');}
\DoxyCodeLine{}
\DoxyCodeLine{var\ combinedStream\ =\ CombinedStream.create();}
\DoxyCodeLine{combinedStream.append(fs.createReadStream('file1.txt'));}
\DoxyCodeLine{combinedStream.append(fs.createReadStream('file2.txt'));}
\DoxyCodeLine{}
\DoxyCodeLine{combinedStream.pipe(fs.createWriteStream('combined.txt'));}

\end{DoxyCode}


While the example above works great, it will pause all source streams until they are needed. If you don\textquotesingle{}t want that to happen, you can set {\ttfamily pause\+Streams} to {\ttfamily false}\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\ javascript}
\DoxyCodeLine{var\ CombinedStream\ =\ require('combined-\/stream');}
\DoxyCodeLine{var\ fs\ =\ require('fs');}
\DoxyCodeLine{}
\DoxyCodeLine{var\ combinedStream\ =\ CombinedStream.create(\{pauseStreams:\ false\});}
\DoxyCodeLine{combinedStream.append(fs.createReadStream('file1.txt'));}
\DoxyCodeLine{combinedStream.append(fs.createReadStream('file2.txt'));}
\DoxyCodeLine{}
\DoxyCodeLine{combinedStream.pipe(fs.createWriteStream('combined.txt'));}

\end{DoxyCode}


However, what if you don\textquotesingle{}t have all the source streams yet, or you don\textquotesingle{}t want to allocate the resources (file descriptors, memory, etc.) for them right away? Well, in that case you can simply provide a callback that supplies the stream by calling a {\ttfamily next()} function\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\ javascript}
\DoxyCodeLine{var\ CombinedStream\ =\ require('combined-\/stream');}
\DoxyCodeLine{var\ fs\ =\ require('fs');}
\DoxyCodeLine{}
\DoxyCodeLine{var\ combinedStream\ =\ CombinedStream.create();}
\DoxyCodeLine{combinedStream.append(function(next)\ \{}
\DoxyCodeLine{\ \ next(fs.createReadStream('file1.txt'));}
\DoxyCodeLine{\});}
\DoxyCodeLine{combinedStream.append(function(next)\ \{}
\DoxyCodeLine{\ \ next(fs.createReadStream('file2.txt'));}
\DoxyCodeLine{\});}
\DoxyCodeLine{}
\DoxyCodeLine{combinedStream.pipe(fs.createWriteStream('combined.txt'));}

\end{DoxyCode}

The effect of those options is described below.\hypertarget{md_node__modules_2combined-stream_2Readme_autotoc_md7862}{}\doxysubsection{\texorpdfstring{combined\+Stream.\+pause\+Streams = $<$tt$>$true$<$/tt$>$}{combinedStream.pauseStreams = <tt>true</tt>}}\label{md_node__modules_2combined-stream_2Readme_autotoc_md7862}
Whether to apply back pressure to the underlaying streams. If set to {\ttfamily false}, the underlaying streams will never be paused. If set to {\ttfamily true}, the underlaying streams will be paused right after being appended, as well as when {\ttfamily delayed\+Stream.\+pipe()} wants to throttle.\hypertarget{md_node__modules_2combined-stream_2Readme_autotoc_md7863}{}\doxysubsection{\texorpdfstring{combined\+Stream.\+max\+Data\+Size = $<$tt$>$2 \texorpdfstring{$\ast$}{*} 1024 \texorpdfstring{$\ast$}{*} 1024$<$/tt$>$}{combinedStream.maxDataSize = <tt>2 * 1024 * 1024</tt>}}\label{md_node__modules_2combined-stream_2Readme_autotoc_md7863}
The maximum amount of bytes (or characters) to buffer for all source streams. If this value is exceeded, {\ttfamily combined\+Stream} emits an `\textquotesingle{}error'\`{} event.\hypertarget{md_node__modules_2combined-stream_2Readme_autotoc_md7864}{}\doxysubsection{\texorpdfstring{combined\+Stream.\+data\+Size = $<$tt$>$0$<$/tt$>$}{combinedStream.dataSize = <tt>0</tt>}}\label{md_node__modules_2combined-stream_2Readme_autotoc_md7864}
The amount of bytes (or characters) currently buffered by {\ttfamily combined\+Stream}.\hypertarget{md_node__modules_2combined-stream_2Readme_autotoc_md7865}{}\doxysubsection{\texorpdfstring{combined\+Stream.\+append(stream)}{combinedStream.append(stream)}}\label{md_node__modules_2combined-stream_2Readme_autotoc_md7865}
Appends the given {\ttfamily stream} to the combined\+Stream object. If {\ttfamily pause\+Streams} is set to \`{}true, this stream will also be paused right away.

{\ttfamily streams} can also be a function that takes one parameter called {\ttfamily next}. {\ttfamily next} is a function that must be invoked in order to provide the {\ttfamily next} stream, see example above.

Regardless of how the {\ttfamily stream} is appended, combined-\/stream always attaches an `\textquotesingle{}error'\`{} listener to it, so you don\textquotesingle{}t have to do that manually.

Special case\+: {\ttfamily stream} can also be a String or Buffer.\hypertarget{md_node__modules_2combined-stream_2Readme_autotoc_md7866}{}\doxysubsection{\texorpdfstring{combined\+Stream.\+write(data)}{combinedStream.write(data)}}\label{md_node__modules_2combined-stream_2Readme_autotoc_md7866}
You should not call this, {\ttfamily combined\+Stream} takes care of piping the appended streams into itself for you.\hypertarget{md_node__modules_2combined-stream_2Readme_autotoc_md7867}{}\doxysubsection{\texorpdfstring{combined\+Stream.\+resume()}{combinedStream.resume()}}\label{md_node__modules_2combined-stream_2Readme_autotoc_md7867}
Causes {\ttfamily combined\+Stream} to start drain the streams it manages. The function is idempotent, and also emits a `\textquotesingle{}resume'\`{} event each time which usually goes to the stream that is currently being drained.\hypertarget{md_node__modules_2combined-stream_2Readme_autotoc_md7868}{}\doxysubsection{\texorpdfstring{combined\+Stream.\+pause();}{combinedStream.pause();}}\label{md_node__modules_2combined-stream_2Readme_autotoc_md7868}
If {\ttfamily combined\+Stream.\+pause\+Streams} is set to {\ttfamily false}, this does nothing. Otherwise a `\textquotesingle{}pause'\`{} event is emitted, this goes to the stream that is currently being drained, so you can use it to apply back pressure.\hypertarget{md_node__modules_2combined-stream_2Readme_autotoc_md7869}{}\doxysubsection{\texorpdfstring{combined\+Stream.\+end();}{combinedStream.end();}}\label{md_node__modules_2combined-stream_2Readme_autotoc_md7869}
