Some projects contain files which are not always meant to be executed in the same environment. For example consider a web application that contains specific code for the server and some specific code for the browser/client. In this case you don’t want to import server-\/only files in your client code.

In order to prevent such scenarios this rule allows you to define restricted zones where you can forbid files from imported if they match a specific path.\hypertarget{md_node__modules_2eslint-plugin-import_2docs_2rules_2no-restricted-paths_autotoc_md10967}{}\doxysection{\texorpdfstring{Rule Details}{Rule Details}}\label{md_node__modules_2eslint-plugin-import_2docs_2rules_2no-restricted-paths_autotoc_md10967}
This rule has one option. The option is an object containing the definition of all restricted {\ttfamily zones} and the optional {\ttfamily base\+Path} which is used to resolve relative paths within. The default value for {\ttfamily base\+Path} is the current working directory.

Each zone consists of the {\ttfamily target} paths, a {\ttfamily from} paths, and an optional {\ttfamily except} and {\ttfamily message} attribute.


\begin{DoxyItemize}
\item {\ttfamily target} contains the paths where the restricted imports should be applied. It can be expressed by
\begin{DoxyItemize}
\item directory string path that matches all its containing files
\item glob pattern matching all the targeted files
\item an array of multiple of the two types above
\end{DoxyItemize}
\item {\ttfamily from} paths define the folders that are not allowed to be used in an import. It can be expressed by
\begin{DoxyItemize}
\item directory string path that matches all its containing files
\item glob pattern matching all the files restricted to be imported
\item an array of multiple directory string path
\item an array of multiple glob patterns
\end{DoxyItemize}
\item {\ttfamily except} may be defined for a zone, allowing exception paths that would otherwise violate the related {\ttfamily from}. Note that it does not alter the behaviour of {\ttfamily target} in any way.
\begin{DoxyItemize}
\item in case {\ttfamily from} contains only glob patterns, {\ttfamily except} must be an array of glob patterns as well
\item in case {\ttfamily from} contains only directory path, {\ttfamily except} is relative to {\ttfamily from} and cannot backtrack to a parent directory
\end{DoxyItemize}
\item {\ttfamily message} -\/ will be displayed in case of the rule violation.
\end{DoxyItemize}\hypertarget{md_node__modules_2eslint-plugin-import_2docs_2rules_2no-restricted-paths_autotoc_md10968}{}\doxysubsection{\texorpdfstring{Examples}{Examples}}\label{md_node__modules_2eslint-plugin-import_2docs_2rules_2no-restricted-paths_autotoc_md10968}
