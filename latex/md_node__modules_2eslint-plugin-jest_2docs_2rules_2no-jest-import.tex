 The {\ttfamily jest} object is automatically in scope within every test file. The methods in the {\ttfamily jest} object help create mocks and let you control Jest\textquotesingle{}s overall behavior. It is therefore completely unnecessary to import in {\ttfamily jest}, as Jest doesn\textquotesingle{}t export anything in the first place.\hypertarget{md_node__modules_2eslint-plugin-jest_2docs_2rules_2no-jest-import_autotoc_md11416}{}\doxysubsection{\texorpdfstring{Rule details}{Rule details}}\label{md_node__modules_2eslint-plugin-jest_2docs_2rules_2no-jest-import_autotoc_md11416}
This rule reports on any importing of Jest.

To name a few\+: `var jest = require(\textquotesingle{}jest');const jest = require(\textquotesingle{}jest\textquotesingle{});{\ttfamily  }import jest from \textquotesingle{}jest\textquotesingle{};import \{jest as test\} from \textquotesingle{}jest\textquotesingle{};\`{}

There is no correct usage of this code, other than to not import {\ttfamily jest} in the first place.\hypertarget{md_node__modules_2eslint-plugin-jest_2docs_2rules_2no-jest-import_autotoc_md11417}{}\doxysection{\texorpdfstring{Further Reading}{Further Reading}}\label{md_node__modules_2eslint-plugin-jest_2docs_2rules_2no-jest-import_autotoc_md11417}
