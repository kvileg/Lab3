This is a \href{http://webpack.js.org/}{\texttt{ webpack}} plugin that simplifies creation of HTML files to serve your {\ttfamily webpack} bundles. This is especially useful for {\ttfamily webpack} bundles that include a hash in the filename which changes every compilation. You can either let the plugin generate an HTML file for you, supply your own template using {\ttfamily lodash} templates or use your own loader.

\doxysubsection*{Usage}

The plugin will generate an HTML5 file for you that includes all your {\ttfamily webpack} bundles in the head using {\ttfamily script} tags. Just add the plugin to your {\ttfamily webpack} config as follows\+:

{\bfseries{webpack.\+config.\+js}} 
\begin{DoxyCode}{0}
\DoxyCodeLine{const\ HtmlWebpackPlugin\ =\ require('html-\/webpack-\/plugin')}
\DoxyCodeLine{}
\DoxyCodeLine{module.exports\ =\ \{}
\DoxyCodeLine{\ \ entry:\ 'index.js',}
\DoxyCodeLine{\ \ output:\ \{}
\DoxyCodeLine{\ \ \ \ path:\ \_\_dirname\ +\ '/dist',}
\DoxyCodeLine{\ \ \ \ filename:\ 'index\_bundle.js'}
\DoxyCodeLine{\ \ \},}
\DoxyCodeLine{\ \ plugins:\ [}
\DoxyCodeLine{\ \ \ \ new\ HtmlWebpackPlugin()}
\DoxyCodeLine{\ \ ]}
\DoxyCodeLine{\}}

\end{DoxyCode}


This will generate a file {\ttfamily dist/index.\+html} containing the following


\begin{DoxyCode}{0}
\DoxyCodeLine{<!DOCTYPE\ html>}
\DoxyCodeLine{<html>}
\DoxyCodeLine{\ \ <head>}
\DoxyCodeLine{\ \ \ \ <meta\ charset="{}utf-\/8"{}>}
\DoxyCodeLine{\ \ \ \ <title>Webpack\ App</title>}
\DoxyCodeLine{\ \ \ \ <script\ defer\ src="{}index\_bundle.js"{}></script>}
\DoxyCodeLine{\ \ </head>}
\DoxyCodeLine{\ \ <body>}
\DoxyCodeLine{\ \ </body>}
\DoxyCodeLine{</html>}

\end{DoxyCode}


If you have multiple {\ttfamily webpack} entry points, they will all be included with {\ttfamily script} tags in the generated HTML.

If you have any CSS assets in webpack\textquotesingle{}s output (for example, CSS extracted with the \href{https://github.com/webpack-contrib/mini-css-extract-plugin}{\texttt{ mini-\/css-\/extract-\/plugin}}) then these will be included with {\ttfamily \texorpdfstring{$<$}{<}link\texorpdfstring{$>$}{>}} tags in the HTML head.

If you have plugins that make use of it, {\ttfamily html-\/webpack-\/plugin} should be ordered first before any of the integrated Plugins.
