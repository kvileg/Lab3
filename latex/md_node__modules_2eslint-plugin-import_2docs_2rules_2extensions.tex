Some file resolve algorithms allow you to omit the file extension within the import source path. For example the {\ttfamily node} resolver (which does not yet support ESM/{\ttfamily import}) can resolve {\ttfamily ./foo/bar} to the absolute path {\ttfamily /\+User/someone/foo/bar.js} because the {\ttfamily .js} extension is resolved automatically by default in CJS. Depending on the resolver you can configure more extensions to get resolved automatically.

In order to provide a consistent use of file extensions across your code base, this rule can enforce or disallow the use of certain file extensions.\hypertarget{md_node__modules_2eslint-plugin-import_2docs_2rules_2extensions_autotoc_md10834}{}\doxysection{\texorpdfstring{Rule Details}{Rule Details}}\label{md_node__modules_2eslint-plugin-import_2docs_2rules_2extensions_autotoc_md10834}
This rule either takes one string option, one object option, or a string and an object option. If it is the string {\ttfamily "{}never"{}} (the default value), then the rule forbids the use for any extension. If it is the string {\ttfamily "{}always"{}}, then the rule enforces the use of extensions for all import statements. If it is the string {\ttfamily "{}ignore\+Packages"{}}, then the rule enforces the use of extensions for all import statements except package imports.


\begin{DoxyCode}{0}
\DoxyCodeLine{"{}import/extensions"{}:\ [<severity>,\ "{}never"{}\ |\ "{}always"{}\ |\ "{}ignorePackages"{}]}

\end{DoxyCode}


By providing an object you can configure each extension separately.


\begin{DoxyCode}{0}
\DoxyCodeLine{"{}import/extensions"{}:\ [<severity>,\ \{}
\DoxyCodeLine{\ \ <extension>:\ "{}never"{}\ |\ "{}always"{}\ |\ "{}ignorePackages"{}}
\DoxyCodeLine{\}]}

\end{DoxyCode}


For example {\ttfamily \{ "{}js"{}\+: "{}always"{}, "{}json"{}\+: "{}never"{} \}} would always enforce the use of the {\ttfamily .js} extension but never allow the use of the {\ttfamily .json} extension.

By providing both a string and an object, the string will set the default setting for all extensions, and the object can be used to set granular overrides for specific extensions.


\begin{DoxyCode}{0}
\DoxyCodeLine{"{}import/extensions"{}:\ [}
\DoxyCodeLine{\ \ <severity>,}
\DoxyCodeLine{\ \ "{}never"{}\ |\ "{}always"{}\ |\ "{}ignorePackages"{},}
\DoxyCodeLine{\ \ \{}
\DoxyCodeLine{\ \ \ \ <extension>:\ "{}never"{}\ |\ "{}always"{}\ |\ "{}ignorePackages"{}}
\DoxyCodeLine{\ \ \}}
\DoxyCodeLine{]}

\end{DoxyCode}


For example, {\ttfamily \mbox{[}"{}error"{}, "{}never"{}, \{ "{}svg"{}\+: "{}always"{} \}\mbox{]}} would require that all extensions are omitted, except for "{}svg"{}.

{\ttfamily ignore\+Packages} can be set as a separate boolean option like this\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{"{}import/extensions"{}:\ [}
\DoxyCodeLine{\ \ <severity>,}
\DoxyCodeLine{\ \ "{}never"{}\ |\ "{}always"{}\ |\ "{}ignorePackages"{},}
\DoxyCodeLine{\ \ \{}
\DoxyCodeLine{\ \ \ \ ignorePackages:\ true\ |\ false,}
\DoxyCodeLine{\ \ \ \ pattern:\ \{}
\DoxyCodeLine{\ \ \ \ \ \ <extension>:\ "{}never"{}\ |\ "{}always"{}\ |\ "{}ignorePackages"{}}
\DoxyCodeLine{\ \ \ \ \}}
\DoxyCodeLine{\ \ \}}
\DoxyCodeLine{]}

\end{DoxyCode}


In that case, if you still want to specify extensions, you can do so inside the {\bfseries{pattern}} property. Default value of {\ttfamily ignore\+Packages} is {\ttfamily false}.\hypertarget{md_node__modules_2eslint-plugin-import_2docs_2rules_2extensions_autotoc_md10835}{}\doxysubsection{\texorpdfstring{Exception}{Exception}}\label{md_node__modules_2eslint-plugin-import_2docs_2rules_2extensions_autotoc_md10835}
When disallowing the use of certain extensions this rule makes an exception and allows the use of extension when the file would not be resolvable without extension.

For example, given the following folder structure\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{├──\ foo}
\DoxyCodeLine{│  \ ├──\ bar.js}
\DoxyCodeLine{│  \ ├──\ bar.json}

\end{DoxyCode}


and this import statement\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{import\ bar\ from\ './foo/bar.json';}

\end{DoxyCode}


then the extension can’t be omitted because it would then resolve to {\ttfamily ./foo/bar.js}.\hypertarget{md_node__modules_2eslint-plugin-import_2docs_2rules_2extensions_autotoc_md10836}{}\doxysubsection{\texorpdfstring{Examples}{Examples}}\label{md_node__modules_2eslint-plugin-import_2docs_2rules_2extensions_autotoc_md10836}

If you are not concerned about a consistent usage of file extension.

In the future, when this rule supports native node ESM resolution, and the plugin is configured to use native rather than transpiled ESM (a config option that is not yet available) -\/ setting this to {\ttfamily always} will have no effect.
