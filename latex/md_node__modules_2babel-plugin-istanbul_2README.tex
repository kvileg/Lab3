A Babel plugin that instruments your code with Istanbul coverage. It can instantly be used with \href{https://github.com/karma-runner/karma-coverage}{\texttt{ karma-\/coverage}} and mocha on Node.\+js (through \href{https://github.com/bcoe/nyc}{\texttt{ nyc}}).

{\bfseries{Note\+:}} This plugin does not generate any report or save any data to any file; it only adds instrumenting code to your Java\+Script source code. To integrate with testing tools, please see the Integrations section.\hypertarget{md_node__modules_2babel-plugin-istanbul_2README_autotoc_md6342}{}\doxysection{\texorpdfstring{Usage}{Usage}}\label{md_node__modules_2babel-plugin-istanbul_2README_autotoc_md6342}

\hypertarget{md_node__modules_2babel-plugin-istanbul_2README_autotoc_md6344}{}\doxysubsection{\texorpdfstring{karma}{karma}}\label{md_node__modules_2babel-plugin-istanbul_2README_autotoc_md6344}
It {\itshape just works} with Karma. First, make sure that the code is already transpiled by Babel (either using {\ttfamily karma-\/babel-\/preprocessor}, {\ttfamily karma-\/webpack}, or {\ttfamily karma-\/browserify}). Then, simply set up \href{https://github.com/karma-runner/karma-coverage}{\texttt{ karma-\/coverage}} according to the docs, but {\bfseries{donâ€™t add the {\ttfamily coverage} preprocessor.}} This plugin has already instrumented your code, and Karma should pick it up automatically.

It has been tested with \href{https://codecov.io/github/bemusic/bemuse}{\texttt{ bemusic/bemuse}} project, which contains \texorpdfstring{$\sim$}{\string~}2400 statements.\hypertarget{md_node__modules_2babel-plugin-istanbul_2README_autotoc_md6345}{}\doxysubsection{\texorpdfstring{mocha on node.\+js (through nyc)}{mocha on node.js (through nyc)}}\label{md_node__modules_2babel-plugin-istanbul_2README_autotoc_md6345}
Configure Mocha to transpile Java\+Script code using Babel, then you can run your tests with \href{https://github.com/bcoe/nyc}{\texttt{ {\ttfamily nyc}}}, which will collect all the coverage report.

babel-\/plugin-\/istanbul respects the {\ttfamily include}/{\ttfamily exclude} configuration options from nyc, but you also need to {\bfseries{configure NYC not to instrument your code}} by adding these settings in your {\ttfamily package.\+json}\+:

You can also use \href{https://github.com/gotwarlost/istanbul/blob/master/ignoring-code-for-coverage.md\#ignoring-code-for-coverage-purposes}{\texttt{ istanbul\textquotesingle{}s ignore hints}} to specify specific lines of code to skip instrumenting.\hypertarget{md_node__modules_2babel-plugin-istanbul_2README_autotoc_md6347}{}\doxysection{\texorpdfstring{Source Maps}{Source Maps}}\label{md_node__modules_2babel-plugin-istanbul_2README_autotoc_md6347}
By default, this plugin will pick up inline source maps and attach them to the instrumented code such that code coverage can be remapped back to the original source, even for multi-\/step build processes. This can be memory intensive. Set {\ttfamily use\+Inline\+Source\+Maps} to prevent this behavior.
