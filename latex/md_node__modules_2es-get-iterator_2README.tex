Get an iterator for any JS language value. Works robustly across all environments, all versions.

In modern engines, {\ttfamily value\mbox{[}Symbol.\+iterator\mbox{]}()} is sufficient to produce an iterator (an object with a {\ttfamily .next} method) for that object. However, older engines\+:
\begin{DoxyItemize}
\item may lack {\ttfamily Symbol} support altogether
\item may have {\ttfamily Symbol.\+iterator} but not implement it on everything it should, like arguments objects
\item may have {\ttfamily Map} and {\ttfamily Set}, but a non-\/standard name for the iterator-\/producing method ({\ttfamily .iterator} or `\mbox{[}'@iterator\textquotesingle{}\mbox{]}{\ttfamily , eg)}
\item {\ttfamily may be old versions of Firefox that produce values until they throw a Stop\+Iteration exception, rather than having iteration result objects}
\item {\ttfamily may be polyfilled/shimmed/shammed, with}es6-\/shim{\ttfamily or}core-\/js\`{} or similar
\end{DoxyItemize}

This library attempts to provide an abstraction over all that complexity!

In node v13+, {\ttfamily exports} is used to provide a lean implementation that lacks all the complexity described above, in combination with the {\ttfamily browser} field so that bundlers will pick up the proper implementation.\hypertarget{md_node__modules_2es-get-iterator_2README_autotoc_md10032}{}\doxysection{\texorpdfstring{Targeting browsers with Symbol support}{Targeting browsers with Symbol support}}\label{md_node__modules_2es-get-iterator_2README_autotoc_md10032}
