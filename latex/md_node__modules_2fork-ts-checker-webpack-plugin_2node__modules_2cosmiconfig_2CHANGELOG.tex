The API has been completely revamped to increase clarity and enable a very wide range of new usage. {\bfseries{Please read the readme for all the details.}}

While the defaults remain just as useful as before — and you can still pass no options at all — now you can also do all kinds of wild and crazy things.


\begin{DoxyItemize}
\item The {\ttfamily loaders} option allows you specify custom functions to derive config objects from files. Your loader functions could parse ES2015 modules or Type\+Script, JSON5, even INI or XML. Whatever suits you.
\item The {\ttfamily search\+Places} option allows you to specify exactly where cosmiconfig looks within each directory it searches.
\item The combination of {\ttfamily loaders} and {\ttfamily search\+Places} means that you should be able to load pretty much any kind of configuration file you want, from wherever you want it to look.
\end{DoxyItemize}

Additionally, the overloaded {\ttfamily load()} function has been split up into several clear and focused functions\+:


\begin{DoxyItemize}
\item {\ttfamily search()} now searches up the directory tree, and {\ttfamily load()} loads a configuration file that you don\textquotesingle{}t need to search for.
\item The {\ttfamily sync} option has been replaced with separate synchronous functions\+: {\ttfamily search\+Sync()} and {\ttfamily load\+Sync()}.
\item {\ttfamily clear\+File\+Cache()} and {\ttfamily clear\+Directory\+Cache()} have been renamed to {\ttfamily clear\+Load\+Cache()} and {\ttfamily clear\+Search\+Path()} respectively.
\end{DoxyItemize}

More details\+:


\begin{DoxyItemize}
\item The default JS loader uses {\ttfamily require}, instead of {\ttfamily require-\/from-\/string}. So you {\itshape could} use {\ttfamily require} hooks to control the loading of JS files (e.\+g. pass them through esm or Babel). In most cases it is probably preferable to use a custom loader.
\item The options {\ttfamily rc}, {\ttfamily js}, and {\ttfamily rc\+Extensions} have all been removed. You can accomplish the same and more with {\ttfamily search\+Places}.
\item The default {\ttfamily search\+Places} include {\ttfamily rc} files with extensions, e.\+g. {\ttfamily .thingrc.\+json}, {\ttfamily .thingrc.\+yaml}, {\ttfamily .thingrc.\+yml}. This is the equivalent of switching the default value of the old {\ttfamily rc\+Extensions} option to {\ttfamily true}.
\item The option {\ttfamily rc\+Strict\+Json} has been removed. To get the same effect, you can specify {\ttfamily no\+Ext\+: cosmiconfig.\+load\+Json} in your {\ttfamily loaders} object.
\item {\ttfamily package\+Prop} no longer accepts {\ttfamily false}. If you don\textquotesingle{}t want to look in {\ttfamily package.\+json}, write a {\ttfamily search\+Places} array that does not include it.
\item By default, empty files are ignored by {\ttfamily search()}. The new option {\ttfamily ignore\+Empty\+Search\+Places} allows you to load them, instead, in case you want to do something with empty files.
\item The option {\ttfamily config\+Path} has been removed. Just pass your filepaths directory to {\ttfamily load()}.
\item Removed the {\ttfamily format} option. Formats are now all handled via the file extensions specified in {\ttfamily loaders}.
\end{DoxyItemize}
