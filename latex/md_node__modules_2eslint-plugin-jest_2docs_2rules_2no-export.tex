 Prevents using {\ttfamily exports} if a file has one or more tests in it.\hypertarget{md_node__modules_2eslint-plugin-jest_2docs_2rules_2no-export_autotoc_md11394}{}\doxysection{\texorpdfstring{Rule Details}{Rule Details}}\label{md_node__modules_2eslint-plugin-jest_2docs_2rules_2no-export_autotoc_md11394}
This rule aims to eliminate duplicate runs of tests by exporting things from test files. If you import from a test file, then all the tests in that file will be run in each imported instance, so bottom line, don\textquotesingle{}t export from a test, but instead move helper functions into a separate file when they need to be shared across tests.

Examples of {\bfseries{incorrect}} code for this rule\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{export\ function\ myHelper()\ \{\}}
\DoxyCodeLine{}
\DoxyCodeLine{module.exports\ =\ function\ ()\ \{\};}
\DoxyCodeLine{}
\DoxyCodeLine{module.exports\ =\ \{}
\DoxyCodeLine{\ \ something:\ 'that\ should\ be\ moved\ to\ a\ non-\/test\ file',}
\DoxyCodeLine{\};}
\DoxyCodeLine{}
\DoxyCodeLine{describe('a\ test',\ ()\ =>\ \{}
\DoxyCodeLine{\ \ expect(1).toBe(1);}
\DoxyCodeLine{\});}

\end{DoxyCode}


Examples of {\bfseries{correct}} code for this rule\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{function\ myHelper()\ \{\}}
\DoxyCodeLine{}
\DoxyCodeLine{const\ myThing\ =\ \{}
\DoxyCodeLine{\ \ something:\ 'that\ can\ live\ here',}
\DoxyCodeLine{\};}
\DoxyCodeLine{}
\DoxyCodeLine{describe('a\ test',\ ()\ =>\ \{}
\DoxyCodeLine{\ \ expect(1).toBe(1);}
\DoxyCodeLine{\});}

\end{DoxyCode}
