Ensures an imported module can be resolved to a module on the local filesystem, as defined by standard Node {\ttfamily require.\+resolve} behavior.

See \href{../../README.md\#settings}{\texttt{ settings}} for customization options for the resolution (i.\+e. additional filetypes, {\ttfamily NODE\+\_\+\+PATH}, etc.)

This rule can also optionally report on unresolved modules in Common\+JS `require('./foo\textquotesingle{}){\ttfamily calls and AMD}require(\mbox{[}\textquotesingle{}./foo\textquotesingle{}\mbox{]}, function (foo) \{...\}){\ttfamily and}define(\mbox{[}\textquotesingle{}./foo\textquotesingle{}\mbox{]}, function (foo) \{...\})\`{}.

To enable this, send {\ttfamily \{ commonjs\+: true/false, amd\+: true/false \}} as a rule option. Both are disabled by default.

\DoxyCodeLine{const\ \{\ default:\ x\ \}\ =\ require('./foo')\ //\ reported\ if\ './foo'\ is\ not\ found}

\hypertarget{md_node__modules_2eslint-plugin-import_2docs_2rules_2no-unresolved_autotoc_md10983}{}\doxysubsubsection{\texorpdfstring{$<$tt$>$ignore$<$/tt$>$}{<tt>ignore</tt>}}\label{md_node__modules_2eslint-plugin-import_2docs_2rules_2no-unresolved_autotoc_md10983}
This rule has its own ignore list, separate from \href{../../README.md\#importignore}{\texttt{ {\ttfamily import/ignore}}}. This is because you may want to know whether a module can be located, regardless of whether it can be parsed for exports\+: {\ttfamily node\+\_\+modules}, Coffee\+Script files, etc. are all good to resolve properly, but will not be parsed if configured as such via \href{../../README.md\#importignore}{\texttt{ {\ttfamily import/ignore}}}.

To suppress errors from files that may not be properly resolved by your \href{../../README.md\#resolver-plugins}{\texttt{ resolver settings}}, you may add an {\ttfamily ignore} key with an array of {\ttfamily Reg\+Exp} pattern strings\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{/*eslint\ import/no-\/unresolved:\ [2,\ \{\ ignore:\ ['\(\backslash\)\(\backslash\).img\$']\ \}]*/}
\DoxyCodeLine{}
\DoxyCodeLine{import\ \{\ x\ \}\ from\ './mod'\ //\ may\ be\ reported,\ if\ not\ resolved\ to\ a\ module}
\DoxyCodeLine{}
\DoxyCodeLine{import\ coolImg\ from\ '../../img/coolImg.img'\ //\ will\ not\ be\ reported,\ even\ if\ not\ found}

\end{DoxyCode}

\DoxyCodeLine{const\ \{\ default:\ x\ \}\ =\ require('./foo')\ //\ reported\ if\ './foo'\ is\ actually\ './Foo'\ and\ caseSensitive:\ true}

\end{DoxyCode}

The {\ttfamily case\+Sensitive} option does not detect case for the current working directory. The {\ttfamily case\+Sensitive\+Strict} option allows checking {\ttfamily cwd} in resolved path. By default, the option is disabled.

\DoxyCodeLine{//\ Absolute\ paths}

\DoxyCodeLine{import\ Foo\ from\ \`{}d:/fOo/bar/file.js`\ //\ reported,\ d:/foo/bar/file.js}
\DoxyCodeLine{}
