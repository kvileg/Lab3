 Jest has built-\/in matchers for expecting equality which allow for more readable tests and error messages if an expectation fails.\hypertarget{md_node__modules_2eslint-plugin-jest_2docs_2rules_2prefer-equality-matcher_autotoc_md11440}{}\doxysection{\texorpdfstring{Rule details}{Rule details}}\label{md_node__modules_2eslint-plugin-jest_2docs_2rules_2prefer-equality-matcher_autotoc_md11440}
This rule checks for {\itshape strict} equality checks ({\ttfamily ===} \& {\ttfamily !==}) in tests that could be replaced with one of the following built-\/in equality matchers\+:


\begin{DoxyItemize}
\item {\ttfamily to\+Be}
\item {\ttfamily to\+Equal}
\item {\ttfamily to\+Strict\+Equal}
\end{DoxyItemize}

Examples of {\bfseries{incorrect}} code for this rule\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{expect(x\ ===\ 5).toBe(true);}
\DoxyCodeLine{expect(name\ ===\ 'Carl').not.toEqual(true);}
\DoxyCodeLine{expect(myObj\ !==\ thatObj).toStrictEqual(true);}

\end{DoxyCode}


Examples of {\bfseries{correct}} code for this rule\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{expect(x).toBe(5);}
\DoxyCodeLine{expect(name).not.toEqual('Carl');}
\DoxyCodeLine{expect(myObj).toStrictEqual(thatObj);}

\end{DoxyCode}
