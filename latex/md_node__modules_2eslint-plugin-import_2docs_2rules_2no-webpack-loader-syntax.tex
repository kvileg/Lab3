Forbid Webpack loader syntax in imports.

\href{https://webpack.js.org}{\texttt{ Webpack}} allows specifying the \href{https://webpack.js.org/concepts/loaders/}{\texttt{ loaders}} to use in the import source string using a special syntax like this\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{var\ moduleWithOneLoader\ =\ require("{}my-\/loader!./my-\/awesome-\/module"{});}

\end{DoxyCode}


This syntax is non-\/standard, so it couples the code to Webpack. The recommended way to specify Webpack loader configuration is in a \href{https://webpack.js.org/concepts/loaders/\#configuration}{\texttt{ Webpack configuration file}}.\hypertarget{md_node__modules_2eslint-plugin-import_2docs_2rules_2no-webpack-loader-syntax_autotoc_md11004}{}\doxysection{\texorpdfstring{Rule Details}{Rule Details}}\label{md_node__modules_2eslint-plugin-import_2docs_2rules_2no-webpack-loader-syntax_autotoc_md11004}
\hypertarget{md_node__modules_2eslint-plugin-import_2docs_2rules_2no-webpack-loader-syntax_autotoc_md11005}{}\doxysubsection{\texorpdfstring{Fail}{Fail}}\label{md_node__modules_2eslint-plugin-import_2docs_2rules_2no-webpack-loader-syntax_autotoc_md11005}

\begin{DoxyCode}{0}
\DoxyCodeLine{import\ myModule\ from\ 'my-\/loader!my-\/module';}
\DoxyCodeLine{import\ theme\ from\ 'style!css!./theme.css';}
\DoxyCodeLine{}
\DoxyCodeLine{var\ myModule\ =\ require('my-\/loader!./my-\/module');}
\DoxyCodeLine{var\ theme\ =\ require('style!css!./theme.css');}

\end{DoxyCode}
