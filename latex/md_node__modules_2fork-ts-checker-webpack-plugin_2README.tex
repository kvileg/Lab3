\doxysection*{Fork TS Checker Webpack Plugin}

 

Webpack plugin that runs Type\+Script type checker on a separate process.



\href{https://www.npmjs.com/package/fork-ts-checker-webpack-plugin}{\texttt{ }} \href{https://github.com/TypeStrong/fork-ts-checker-webpack-plugin/actions?query=branch\%3Amain+event\%3Apush}{\texttt{ }} \href{https://npmjs.org/package/fork-ts-checker-webpack-plugin}{\texttt{ }} \href{http://commitizen.github.io/cz-cli/}{\texttt{ }} \href{https://github.com/prettier/prettier}{\texttt{ }} \href{https://github.com/semantic-release/semantic-release}{\texttt{ }}

\hypertarget{md_node__modules_2fork-ts-checker-webpack-plugin_2README_autotoc_md14190}{}\doxysubsection{\texorpdfstring{Features}{Features}}\label{md_node__modules_2fork-ts-checker-webpack-plugin_2README_autotoc_md14190}

\begin{DoxyItemize}
\item Speeds up \href{https://github.com/Microsoft/TypeScript}{\texttt{ Type\+Script}} type checking and \href{https://eslint.org/}{\texttt{ ESLint}} linting (by moving each to a separate process) 🏎
\item Supports modern Type\+Script features like \href{https://www.typescriptlang.org/docs/handbook/project-references.html}{\texttt{ project references}} and \href{https://www.typescriptlang.org/docs/handbook/release-notes/typescript-3-4.html\#faster-subsequent-builds-with-the---incremental-flag}{\texttt{ incremental mode}} ✨
\item Supports \href{https://vuejs.org/v2/guide/single-file-components.html}{\texttt{ Vue Single File Component}} ✅ 
\item Displays nice error messages with the \href{https://babeljs.io/docs/en/next/babel-code-frame.html}{\texttt{ code frame}} formatter 🌈
\end{DoxyItemize}\hypertarget{md_node__modules_2fork-ts-checker-webpack-plugin_2README_autotoc_md14191}{}\doxysubsection{\texorpdfstring{Installation}{Installation}}\label{md_node__modules_2fork-ts-checker-webpack-plugin_2README_autotoc_md14191}

\hypertarget{md_node__modules_2fork-ts-checker-webpack-plugin_2README_autotoc_md14192}{}\doxysubsection{\texorpdfstring{Modules resolution}{Modules resolution}}\label{md_node__modules_2fork-ts-checker-webpack-plugin_2README_autotoc_md14192}
It\textquotesingle{}s very important to be aware that {\bfseries{this plugin uses \href{https://github.com/Microsoft/TypeScript}{\texttt{ Type\+Script}}\textquotesingle{}s, not \href{https://github.com/webpack/webpack}{\texttt{ webpack}}\textquotesingle{}s modules resolution}}. It means that you have to setup {\ttfamily tsconfig.\+json} correctly.

\begin{quote}
It\textquotesingle{}s because of the performance -\/ with Type\+Script\textquotesingle{}s module resolution we don\textquotesingle{}t have to wait for webpack to compile files.

To debug Type\+Script\textquotesingle{}s modules resolution, you can use {\ttfamily tsc -\/-\/trace\+Resolution} command. \end{quote}

\begin{DoxyItemize}
\item {\ttfamily "{}fork-\/ts-\/checker"{}} field in the {\ttfamily package.\+json}
\item {\ttfamily .fork-\/ts-\/checkerrc} file in JSON or YAML format
\item {\ttfamily fork-\/ts-\/checker.\+config.\+js} file exporting a JS object
\end{DoxyItemize}

Options passed to the plugin constructor will overwrite options from the cosmiconfig (using \href{https://github.com/TehShrike/deepmerge}{\texttt{ deepmerge}}).
