This package has two modes that are used depending on the environment and parameters of use.\hypertarget{md_node__modules_2_0dnodelib_2fs_8scandir_2README_autotoc_md1940}{}\doxysubsection{\texorpdfstring{old}{old}}\label{md_node__modules_2_0dnodelib_2fs_8scandir_2README_autotoc_md1940}

\begin{DoxyItemize}
\item Node.\+js below {\ttfamily 10.\+10} or when the {\ttfamily stats} option is enabled
\end{DoxyItemize}

When working in the old mode, the directory is read first ({\ttfamily fs.\+readdir}), then the type of entries is determined ({\ttfamily fs.\+lstat} and/or {\ttfamily fs.\+stat} for symbolic links).\hypertarget{md_node__modules_2_0dnodelib_2fs_8scandir_2README_autotoc_md1941}{}\doxysubsection{\texorpdfstring{modern}{modern}}\label{md_node__modules_2_0dnodelib_2fs_8scandir_2README_autotoc_md1941}

\begin{DoxyItemize}
\item Node.\+js 10.\+10+ and the {\ttfamily stats} option is disabled
\end{DoxyItemize}

In the modern mode, reading the directory ({\ttfamily fs.\+readdir} with the {\ttfamily with\+File\+Types} option) is combined with obtaining information about its entries. An additional call for symbolic links ({\ttfamily fs.\+stat}) is still present.

This mode makes fewer calls to the file system. It\textquotesingle{}s faster.\hypertarget{md_node__modules_2_0dnodelib_2fs_8scandir_2README_autotoc_md1942}{}\doxysection{\texorpdfstring{Changelog}{Changelog}}\label{md_node__modules_2_0dnodelib_2fs_8scandir_2README_autotoc_md1942}
