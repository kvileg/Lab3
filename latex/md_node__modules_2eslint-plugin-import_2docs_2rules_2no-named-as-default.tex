Reports use of an exported name as the locally imported name of a default export.

Rationale\+: using an exported name as the name of the default export is likely...


\begin{DoxyItemize}
\item {\itshape misleading}\+: others familiar with {\ttfamily foo.\+js} probably expect the name to be {\ttfamily foo}
\item {\itshape a mistake}\+: only needed to import {\ttfamily bar} and forgot the brackets (the case that is prompting this)
\end{DoxyItemize}\hypertarget{md_node__modules_2eslint-plugin-import_2docs_2rules_2no-named-as-default_autotoc_md10945}{}\doxysection{\texorpdfstring{Rule Details}{Rule Details}}\label{md_node__modules_2eslint-plugin-import_2docs_2rules_2no-named-as-default_autotoc_md10945}
Given\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{//\ foo.js}
\DoxyCodeLine{export\ default\ 'foo';}
\DoxyCodeLine{export\ const\ bar\ =\ 'baz';}

\end{DoxyCode}


...this would be valid\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{import\ foo\ from\ './foo.js';}

\end{DoxyCode}


...and this would be reported\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{//\ message:\ Using\ exported\ name\ 'bar'\ as\ identifier\ for\ default\ export.}
\DoxyCodeLine{import\ bar\ from\ './foo.js';}

\end{DoxyCode}


For post-\/\+ES2015 {\ttfamily export} extensions, this also prevents exporting the default from a referenced module as a name within that module, for the same reasons\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{//\ valid:}
\DoxyCodeLine{export\ foo\ from\ './foo.js';}
\DoxyCodeLine{}
\DoxyCodeLine{//\ message:\ Using\ exported\ name\ 'bar'\ as\ identifier\ for\ default\ export.}
\DoxyCodeLine{export\ bar\ from\ './foo.js';}

\end{DoxyCode}
