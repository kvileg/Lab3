 {\bfseries{A small, fast Java\+Script library for arbitrary-\/precision decimal arithmetic.}}

The little sister to \href{https://github.com/MikeMcl/bignumber.js/}{\texttt{ bignumber.\+js}} and \href{https://github.com/MikeMcl/decimal.js/}{\texttt{ decimal.\+js}}. See \href{https://github.com/MikeMcl/big.js/wiki}{\texttt{ here}} for some notes on the difference between them.\hypertarget{md_node__modules_2big_8js_2README_autotoc_md6585}{}\doxysection{\texorpdfstring{Features}{Features}}\label{md_node__modules_2big_8js_2README_autotoc_md6585}

\begin{DoxyItemize}
\item Faster, smaller and easier-\/to-\/use than Java\+Script versions of Java\textquotesingle{}s Big\+Decimal
\item Only 5.\+9 KB minified and 2.\+7 KB gzipped
\item Simple API
\item Replicates the {\ttfamily to\+Exponential}, {\ttfamily to\+Fixed} and {\ttfamily to\+Precision} methods of Java\+Script\textquotesingle{}s Number type
\item Includes a {\ttfamily sqrt} method
\item Stores values in an accessible decimal floating point format
\item No dependencies
\item Comprehensive \href{http://mikemcl.github.io/big.js/}{\texttt{ documentation}} and test set
\end{DoxyItemize}\hypertarget{md_node__modules_2big_8js_2README_autotoc_md6586}{}\doxysection{\texorpdfstring{Set up}{Set up}}\label{md_node__modules_2big_8js_2README_autotoc_md6586}

 \hypertarget{md_node__modules_2big_8js_2README_autotoc_md6587}{}\doxysection{\texorpdfstring{Use}{Use}}\label{md_node__modules_2big_8js_2README_autotoc_md6587}
{\itshape In all examples below, {\ttfamily var}, semicolons and {\ttfamily to\+String} calls are not shown. If a commented-\/out value is in quotes it means {\ttfamily to\+String} has been called on the preceding expression.}

The library exports a single function, {\ttfamily Big}, the constructor of Big number instances. It accepts a value of type number, string or Big number object. \begin{DoxyVerb}x = new Big(123.4567)
y = Big('123456.7e-3')             // 'new' is optional
z = new Big(x)
x.eq(y) && x.eq(z) && y.eq(z)      // true
\end{DoxyVerb}
 A Big number is immutable in the sense that it is not changed by its methods. \begin{DoxyVerb}0.3 - 0.1                          // 0.19999999999999998
x = new Big(0.3)
x.minus(0.1)                       // "0.2"
x                                  // "0.3"
\end{DoxyVerb}
 The methods that return a Big number can be chained. \begin{DoxyVerb}x.div(y).plus(z).times(9).minus('1.234567801234567e+8').plus(976.54321).div('2598.11772')
x.sqrt().div(y).pow(3).gt(y.mod(z))    // true
\end{DoxyVerb}
 Like Java\+Script\textquotesingle{}s Number type, there are {\ttfamily to\+Exponential}, {\ttfamily to\+Fixed} and {\ttfamily to\+Precision} methods. \begin{DoxyVerb}x = new Big(255.5)
x.toExponential(5)                 // "2.55500e+2"
x.toFixed(5)                       // "255.50000"
x.toPrecision(5)                   // "255.50"
\end{DoxyVerb}
 The arithmetic methods always return the exact result except {\ttfamily div}, {\ttfamily sqrt} and {\ttfamily pow} (with negative exponent), as these methods involve division.

The maximum number of decimal places and the rounding mode used to round the results of these methods is determined by the value of the {\ttfamily DP} and {\ttfamily RM} properties of the {\ttfamily Big} number constructor. \begin{DoxyVerb}Big.DP = 10
Big.RM = 1

x = new Big(2);
y = new Big(3);
z = x.div(y)                       // "0.6666666667"
z.sqrt()                           // "0.8164965809"
z.pow(-3)                          // "3.3749999995"
z.times(z)                         // "0.44444444448888888889"
z.times(z).round(10)               // "0.4444444445"
\end{DoxyVerb}
 Multiple Big number constructors can be created, each with an independent configuration.

The value of a Big number is stored in a decimal floating point format in terms of a coefficient, exponent and sign. \begin{DoxyVerb}x = new Big(-123.456);
x.c                                // [1,2,3,4,5,6]    coefficient (i.e. significand)
x.e                                // 2                exponent
x.s                                // -1               sign
\end{DoxyVerb}
 For further information see the \href{http://mikemcl.github.io/big.js/}{\texttt{ API}} reference from the {\itshape doc} folder.\hypertarget{md_node__modules_2big_8js_2README_autotoc_md6588}{}\doxysection{\texorpdfstring{Test}{Test}}\label{md_node__modules_2big_8js_2README_autotoc_md6588}
The {\itshape test} directory contains the test scripts for each Big number method.

The tests can be run with Node.\+js or a browser.

To run all the tests \begin{DoxyVerb}$ npm test
\end{DoxyVerb}
 To test a single method \begin{DoxyVerb}$ node test/toFixed
\end{DoxyVerb}
 For the browser, see {\itshape single-\/test.\+html} and {\itshape every-\/test.\+html} in the {\itshape test/browser} directory.

{\itshape big-\/vs-\/number.\+html} is a simple application that enables some of the methods of big.\+js to be compared with those of Java\+Script\textquotesingle{}s Number type.\hypertarget{md_node__modules_2big_8js_2README_autotoc_md6589}{}\doxysection{\texorpdfstring{Performance}{Performance}}\label{md_node__modules_2big_8js_2README_autotoc_md6589}
The {\itshape perf} directory contains two legacy applications and a {\itshape lib} directory containing the Big\+Decimal libraries used by both.

{\itshape big-\/vs-\/bigdecimal.\+html} tests the performance of big.\+js against the Java\+Script translations of two versions of Big\+Decimal, its use should be more or less self-\/explanatory.


\begin{DoxyItemize}
\item \href{https://github.com/iriscouch/bigdecimal.js}{\texttt{ GWT\+: java.\+math.\+Big\+Decimal}}
\item \href{https://github.com/dtrebbien/BigDecimal.js}{\texttt{ ICU4\+J\+: com.\+ibm.\+icu.\+math.\+Big\+Decimal}}
\end{DoxyItemize}

The Big\+Decimal in the npm registry is the GWT version. It has some bugs, see the Node.\+js script {\itshape perf/lib/bigdecimal\+\_\+\+GWT/bugs.\+js} for examples of flaws in its {\itshape remainder}, {\itshape divide} and {\itshape compare\+To} methods.

{\itshape bigtime.\+js} is a Node.\+js command-\/line application which tests the performance of big.\+js against the GWT version of Big\+Decimal from the npm registry.

For example, to compare the time taken by the big.\+js {\ttfamily plus} method and the Big\+Decimal {\ttfamily add} method \begin{DoxyVerb}$ node bigtime plus 10000 40
\end{DoxyVerb}
 This will time 10000 calls to each, using operands of up to 40 random digits and will check that the results match.

For help \begin{DoxyVerb}$ node bigtime -h
\end{DoxyVerb}
