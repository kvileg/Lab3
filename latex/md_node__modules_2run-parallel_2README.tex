\hypertarget{md_node__modules_2run-parallel_2README_autotoc_md25000}{}\doxysubsubsection{\texorpdfstring{parallel(tasks, \mbox{[}callback\mbox{]})}{parallel(tasks, [callback])}}\label{md_node__modules_2run-parallel_2README_autotoc_md25000}
Run the {\ttfamily tasks} array of functions in parallel, without waiting until the previous function has completed. If any of the functions pass an error to its callback, the main {\ttfamily callback} is immediately called with the value of the error. Once the {\ttfamily tasks} have completed, the results are passed to the final {\ttfamily callback} as an array.

It is also possible to use an object instead of an array. Each property will be run as a function and the results will be passed to the final {\ttfamily callback} as an object instead of an array. This can be a more readable way of handling the results.\hypertarget{md_node__modules_2run-parallel_2README_autotoc_md25001}{}\doxyparagraph{\texorpdfstring{arguments}{arguments}}\label{md_node__modules_2run-parallel_2README_autotoc_md25001}

\begin{DoxyItemize}
\item {\ttfamily tasks} -\/ An array or object containing functions to run. Each function is passed a {\ttfamily callback(err, result)} which it must call on completion with an error {\ttfamily err} (which can be {\ttfamily null}) and an optional {\ttfamily result} value.
\item {\ttfamily callback(err, results)} -\/ An optional callback to run once all the functions have completed. This function gets a results array (or object) containing all the result arguments passed to the task callbacks.
\end{DoxyItemize}\hypertarget{md_node__modules_2run-parallel_2README_autotoc_md25002}{}\doxyparagraph{\texorpdfstring{example}{example}}\label{md_node__modules_2run-parallel_2README_autotoc_md25002}
