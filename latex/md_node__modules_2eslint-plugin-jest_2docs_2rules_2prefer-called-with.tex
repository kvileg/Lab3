 The {\ttfamily to\+Be\+Called()} matcher is used to assert that a mock function has been called one or more times, without checking the arguments passed. The assertion is stronger when arguments are also validated using the {\ttfamily to\+Be\+Called\+With()} matcher. When some arguments are difficult to check, using generic match like {\ttfamily expect.\+anything()} at least enforces number and position of arguments.

This rule warns if the form without argument checking is used, except for {\ttfamily .not} enforcing a function has never been called.\hypertarget{md_node__modules_2eslint-plugin-jest_2docs_2rules_2prefer-called-with_autotoc_md11436}{}\doxysection{\texorpdfstring{Rule details}{Rule details}}\label{md_node__modules_2eslint-plugin-jest_2docs_2rules_2prefer-called-with_autotoc_md11436}
The following patterns are warnings\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{expect(someFunction).toBeCalled();}
\DoxyCodeLine{}
\DoxyCodeLine{expect(someFunction).toHaveBeenCalled();}

\end{DoxyCode}


The following patterns are not warnings\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{expect(noArgsFunction).toBeCalledWith();}
\DoxyCodeLine{}
\DoxyCodeLine{expect(roughArgsFunction).toBeCalledWith(expect.anything(),\ expect.any(Date));}
\DoxyCodeLine{}
\DoxyCodeLine{expect(anyArgsFunction).toBeCalledTimes(1);}
\DoxyCodeLine{}
\DoxyCodeLine{expect(uncalledFunction).not.toBeCalled();}

\end{DoxyCode}
