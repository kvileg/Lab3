 Conditional logic in tests is usually an indication that a test is attempting to cover too much, and not testing the logic it intends to. Each branch of code executing within an if statement will usually be better served by a test devoted to it.

Conditionals are often used to satisfy the typescript type checker. In these cases, using the non-\/null assertion operator (!) would be best.\hypertarget{md_node__modules_2eslint-plugin-jest_2docs_2rules_2no-if_autotoc_md11407}{}\doxysection{\texorpdfstring{Rule Details}{Rule Details}}\label{md_node__modules_2eslint-plugin-jest_2docs_2rules_2no-if_autotoc_md11407}
This rule prevents the use of if/ else statements and conditional (ternary) operations in tests.

The following patterns are considered warnings\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{it('foo',\ ()\ =>\ \{}
\DoxyCodeLine{\ \ if\ ('bar')\ \{}
\DoxyCodeLine{\ \ \ \ //\ an\ if\ statement\ here\ is\ invalid}
\DoxyCodeLine{\ \ \ \ //\ you\ are\ probably\ testing\ too\ much}
\DoxyCodeLine{\ \ \}}
\DoxyCodeLine{\});}
\DoxyCodeLine{}
\DoxyCodeLine{it('foo',\ ()\ =>\ \{}
\DoxyCodeLine{\ \ const\ bar\ =\ foo\ ?\ 'bar'\ :\ null;}
\DoxyCodeLine{\});}

\end{DoxyCode}


These patterns would not be considered warnings\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{it('foo',\ ()\ =>\ \{}
\DoxyCodeLine{\ \ //\ only\ test\ the\ 'foo'\ case}
\DoxyCodeLine{\});}
\DoxyCodeLine{}
\DoxyCodeLine{it('bar',\ ()\ =>\ \{}
\DoxyCodeLine{\ \ //\ test\ the\ 'bar'\ case\ separately}
\DoxyCodeLine{\});}
\DoxyCodeLine{}
\DoxyCodeLine{it('foo',\ ()\ =>\ \{}
\DoxyCodeLine{\ \ function\ foo(bar)\ \{}
\DoxyCodeLine{\ \ \ \ //\ nested\ functions\ are\ valid}
\DoxyCodeLine{\ \ \ \ return\ foo\ ?\ bar\ :\ null;}
\DoxyCodeLine{\ \ \}}
\DoxyCodeLine{\});}

\end{DoxyCode}
\hypertarget{md_node__modules_2eslint-plugin-jest_2docs_2rules_2no-if_autotoc_md11408}{}\doxysection{\texorpdfstring{When Not To Use It}{When Not To Use It}}\label{md_node__modules_2eslint-plugin-jest_2docs_2rules_2no-if_autotoc_md11408}
If you do not wish to prevent the use of if statements in tests, you can safely disable this rule.
