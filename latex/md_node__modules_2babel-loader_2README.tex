\doxysection*{Babel Loader}

This package allows transpiling Java\+Script files using \href{https://github.com/babel/babel}{\texttt{ Babel}} and \href{https://github.com/webpack/webpack}{\texttt{ webpack}}.

{\bfseries{Note}}\+: Issues with the output should be reported on the Babel \href{https://github.com/babel/babel/issues}{\texttt{ Issues}} tracker.

This loader also supports the following loader-\/specific option\+:


\begin{DoxyItemize}
\item {\ttfamily cache\+Directory}\+: Default {\ttfamily false}. When set, the given directory will be used to cache the results of the loader. Future webpack builds will attempt to read from the cache to avoid needing to run the potentially expensive Babel recompilation process on each run. If the value is set to {\ttfamily true} in options ({\ttfamily \{cache\+Directory\+: true\}}), the loader will use the default cache directory in {\ttfamily node\+\_\+modules/.cache/babel-\/loader} or fallback to the default OS temporary file directory if no {\ttfamily node\+\_\+modules} folder could be found in any root directory.
\item {\ttfamily cache\+Identifier}\+: Default is a string composed by the {\ttfamily @babel/core}\textquotesingle{}s version, the {\ttfamily babel-\/loader}\textquotesingle{}s version, the contents of {\ttfamily .babelrc} file if it exists, and the value of the environment variable {\ttfamily BABEL\+\_\+\+ENV} with a fallback to the {\ttfamily NODE\+\_\+\+ENV} environment variable. This can be set to a custom value to force cache busting if the identifier changes.
\item {\ttfamily cache\+Compression}\+: Default {\ttfamily true}. When set, each Babel transform output will be compressed with Gzip. If you want to opt-\/out of cache compression, set it to {\ttfamily false} -- your project may benefit from this if it transpiles thousands of files.
\item {\ttfamily customize}\+: Default {\ttfamily null}. The path of a module that exports a {\ttfamily custom} callback like the one that you\textquotesingle{}d pass to \`{}.custom()\`{}. Since you already have to make a new file to use this, it is recommended that you instead use {\ttfamily .custom} to create a wrapper loader. Only use this if you {\itshape must} continue using {\ttfamily babel-\/loader} directly, but still want to customize.
\item {\ttfamily metadata\+Subscribers}\+: Default {\ttfamily \mbox{[}\mbox{]}}. Takes an array of context function names. E.\+g. if you passed \mbox{[}\textquotesingle{}my\+Metadata\+Plugin\textquotesingle{}\mbox{]}, you\textquotesingle{}d assign a subscriber function to {\ttfamily context.\+my\+Metadata\+Plugin} within your webpack plugin\textquotesingle{}s hooks \& that function will be called with {\ttfamily metadata}.
\end{DoxyItemize}\hypertarget{md_node__modules_2babel-loader_2README_autotoc_md6260}{}\doxysubsection{\texorpdfstring{Troubleshooting}{Troubleshooting}}\label{md_node__modules_2babel-loader_2README_autotoc_md6260}

Make sure you are transforming as few files as possible. Because you are probably matching {\ttfamily /\textbackslash{}.m?js\$/}, you might be transforming the {\ttfamily node\+\_\+modules} folder or other unwanted source.

To exclude {\ttfamily node\+\_\+modules}, see the {\ttfamily exclude} option in the {\ttfamily loaders} config as documented above.

You can also speed up babel-\/loader by as much as 2x by using the {\ttfamily cache\+Directory} option. This will cache transformations to the filesystem.\hypertarget{md_node__modules_2babel-loader_2README_autotoc_md6262}{}\doxysubsubsection{\texorpdfstring{Some files in my node\+\_\+modules are not transpiled for IE 11}{Some files in my node\_modules are not transpiled for IE 11}}\label{md_node__modules_2babel-loader_2README_autotoc_md6262}
