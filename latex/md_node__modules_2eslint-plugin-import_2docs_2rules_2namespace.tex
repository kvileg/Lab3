Enforces names exist at the time they are dereferenced, when imported as a full namespace (i.\+e. `import \texorpdfstring{$\ast$}{*} as foo from './foo\textquotesingle{}; foo.\+bar();{\ttfamily will report if}bar{\ttfamily is not exported by}./foo\`{}.).

Will report at the import declaration if there are {\itshape no} exported names found.

Also, will report for computed references (i.\+e. {\ttfamily foo\mbox{[}"{}bar"{}\mbox{]}()}).

Reports on assignment to a member of an imported namespace.

Note\+: for packages, the plugin will find exported names from \href{https://github.com/rollup/rollup/wiki/jsnext:main}{\texttt{ {\ttfamily jsnext\+:main}}}, if present in {\ttfamily package.\+json}. Redux\textquotesingle{}s npm module includes this key, and thereby is lintable, for example.

A module path that is \href{../README.md\#importignore}{\texttt{ ignored}} or not \href{https://github.com/bmeck/UnambiguousJavaScriptGrammar}{\texttt{ unambiguously an ES module}} will not be reported when imported.\hypertarget{md_node__modules_2eslint-plugin-import_2docs_2rules_2namespace_autotoc_md10865}{}\doxysection{\texorpdfstring{Rule Details}{Rule Details}}\label{md_node__modules_2eslint-plugin-import_2docs_2rules_2namespace_autotoc_md10865}
Currently, this rule does not check for possible redefinition of the namespace in an intermediate scope. Adherence to the ESLint {\ttfamily no-\/shadow} rule for namespaces will prevent this from being a problem.

For \href{https://github.com/leebyron/ecmascript-more-export-from}{\texttt{ ES7}}, reports if an exported namespace would be empty (no names exported from the referenced module.)
